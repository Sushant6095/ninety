# AUDIT — 2026-07-14

Branch `merge/live-integration`. Read-only. Every claim below traces to a file, a commit sha, or a command that was run.

**Method.** 30 read-only agents: 18 commit auditors (message vs diff), 8 mandate auditors, 7 state auditors, plus direct verification by the lead auditor. The full test suite was run once. The dev server was probed with curl + headless Playwright. Nothing was edited.

**Caveats that bound everything below.**

1. A concurrent agent was editing `apps/web/src` throughout. HEAD moved **1c32480 → a463303 → c2a6c94 → ebed784** during the audit; ADR-055 and ADR-056 appeared mid-run. Committed state was read via `git show HEAD:<path>`; where the working tree diverges it is called out.
2. `.claude/trace/` **cannot evidence tool usage.** Its PostToolUse matcher is literally `Write|Edit|MultiEdit|Bash` (`.claude/settings.json`), so across 3,014 records the `tool` field is only ever Bash (58.2%) / Edit (24.1%) / Write (17.7%) — **zero** MCP, Skill, or Task records. The project's claim that hooks "trace every action" is false. Every `mcp__` and skill-name string in the trace is a Bash command or a file write, including this audit's own greps. Mandate verdicts were therefore taken from the **session transcripts** (`~/.claude/projects/…/*.jsonl`, 18 files, 264 MB), which do record `mcp__*` tool calls, plus artifacts on disk.
