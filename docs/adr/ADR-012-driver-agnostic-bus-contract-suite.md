# ADR 012 driver-agnostic bus contract suite

Extends ADR-007. Status: accepted 2026-07-06. Context: ADR-007 committed to one `Bus` interface with two drivers (Redis Streams now, Kafka on graduation), but nothing enforced that a second driver behaves like the first — a silent semantic drift on the BUS_DRIVER flip would corrupt event ordering or delivery. Decision: bus-driver interchangeability is enforced by a driver-agnostic contract suite — `packages/bus/src/contract.ts`, `busContract(name, harness)` — asserting the three behaviours every driver must uphold: order-per-key, at-least-once delivery (an entry unacked at crash time is redelivered), and ack semantics (an acked entry is never redelivered). A driver plugs in via a `ContractHarness` (makeBus / makeTopic / available / per-driver opts, optional pending introspection); the same file carries compile-time `Conforms<Driver>` proofs so a driver that stops implementing `Bus` fails typecheck. A new driver MUST pass the contract (and typecheck) before BUS_DRIVER may point at it. Consequences: (+) RedisBus passes the contract against docker redis today; KafkaBus conforms at compile time but throws at runtime until kafkajs is wired, at which point it runs the SAME contract with a Kafka harness — no consumer changes. (+) the flip becomes a gated, test-backed operation rather than a leap of faith. (−) the crash/recovery timing knobs (blockMs, minIdleMs) are supplied per-harness, so a driver with a fundamentally different recovery model may need harness-level accommodation. Cross-ref: ADR-007 (the interface + BUS_DRIVER flip, amended by this ADR), ADR-011 (RedisBus delivery semantics this contract pins down).
