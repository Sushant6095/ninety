# ADR 020 system signal plane through the bus

Supersedes-in-part ADR-019 (2026-07-07): replaces its proposed `feed.gap` carve-out. Status: accepted 2026-07-07. Context: ADR-019 needed to emit `feed.gap` ops alerts but the typed `AnyEvent` bus has no system/fixture variant, so it shipped a raw `redis.xadd("feed.gap.v1")` side-stream OUTSIDE `packages/bus` and PROPOSED a CLAUDE.md carve-out bullet blessing such raw system streams. That carve-out weakened the Architecture law's single invariant — all comms through `packages/bus` — by permitting raw Redis side channels, and it would grow one exception per future signal. This ADR replaces the carve-out with a second first-class PLANE that still runs through the bus, so the law keeps ONE communication path. Both planes are implemented and gate-green. Decision: (1) `packages/bus` now carries TWO planes, both through the bus with no raw side channels — DOMAIN events (`AnyEvent`, existing topics) and SYSTEM signals (`SysEvent`, `sys.*` topics). (2) `SysEvent` (`packages/schema/src/sys.ts`) is a `z.discriminatedUnion("kind", …)` over a shared header `{sig_id: uuid, ts, severity: info|warn|crit}`; kinds `feed_gap | backpressure | saga_stuck | replay_request`; topic `sys.signals.v1`. It is DELIBERATELY not part of `AnyEvent` and carries NO `(source, source_seq)` / `match_id` idempotency — signals are ops/admin traffic, not market data, so replay/dedup/partitioning don't apply. (3) the `Bus` interface is now generic AND topic-bound: `PayloadOf<T>` maps `sys.*` topics → `SysEvent` and every other topic → `Envelope` via a NON-distributive `[T] extends […]` conditional, so `publish`/`consume` infer the right payload per topic and a wrong-plane handler FAILS TO COMPILE. This closed a HIGH type-safety finding: a sys consumer typed against `Envelope` would otherwise silently never match, since `SysEvent` discriminates on `kind`, not `type`. Both drivers (Redis + Kafka stub) were updated and it survives the `BUS_DRIVER=kafka` flip; a compile-time regression guard lives in `packages/bus/src/contract.ts`. (4) `feed.gap` MIGRATED off the raw `redis.xadd("feed.gap.v1")` side-stream onto `bus.publish(TOPICS.sysSignals, feedGapSignal(info))`; the raw stream and the `FEED_GAP_STREAM` constant are DELETED. Consequences: (+) the Architecture law keeps ONE path — all inter-service comms flow through `packages/bus`; the proposed raw-stream carve-out is gone and future signals add a `kind`, not an exception. (+) the type system makes cross-plane misuse a compile error, not a silent no-match. (+) VERIFY green: a gap-simulation test consumes `feed.gap` THROUGH the bus, and a grep proves zero raw stream access (`xadd`/`xreadgroup`/`xgroup`/`xautoclaim`/`xack`) outside `packages/bus`. (~) system signals intentionally forgo idempotency/replay/partitioning; a dropped signal is an ops miss, not market-data loss — do not route domain state through `sys.*`. (−) `SysEvent` is a second schema union to keep in sync with `AnyEvent`; the `contract.ts` compile-guard + topic-bound `PayloadOf` are the mitigation. CLAUDE.md governance: the Architecture law's "Events only…" line was AMENDED this session (verbatim, human-authorized — not proposed) to "ALL inter-service communication flows through packages/bus. Two planes: domain events (AnyEvent, existing topics) and system signals (SysEvent, sys.* topics). Raw Redis/Kafka access is forbidden outside packages/bus — with two named intra-service exceptions that are storage, not communication: the engine journal and WS resume buffers." Nuance worth recording: worker-ingest's `fixtures:current` compacted hash (`HSET`/`HGET`) is KV STORAGE, not a stream/comms channel, so it is consistent with the law's intent — the two named exceptions cover stream-shaped storage; plain KV/SQL storage was never inter-service communication and needs no exception. Play-money invariant unaffected; NO admin/result path added. Cross-refs ADR-019 (feed.gap gap recovery, carve-out superseded here), ADR-018 (ingest normalization), ADR-010 (AnyEvent typed contract), ADR-011 (bus delivery semantics), ADR-007 (bus Redis→Kafka).
