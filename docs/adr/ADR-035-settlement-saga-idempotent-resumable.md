# ADR 035 settlement saga — 7-step idempotent, persisted, resumable; credit winners qty×100; post matchday root; stuck alerts (apps/worker-jobs/src/settlement.ts, prompt 24)

Status: accepted 2026-07-08. Context: prompt 24 — orchestrate on-chain settlement as a crash-safe saga: `FT_DETECTED → PROOF_FETCHED → TX_SENT → CONFIRMED → CREDITED → ROOT_POSTED → MOMENTS → DONE`, each step idempotent and persisted (SettlementSaga), resumable after a worker crash with ZERO double-credit; credit winners qty×100 via the ledger; compute + post the matchday leaderboard root; alert if a saga stalls >120s. The worker FORWARDS the proof; `programs/omnipitch_core/proof.rs` VERIFIES it — no admin result path. Decision (implemented, gate 9/9/9, 7 saga tests): a pure state machine (`runSettlement`) over a `SagaRecord {marketId, matchday, step, result?, proof?, sig?, rootSig?, updatedAt}`, with ALL side effects behind an injected `SagaDeps` (loadSaga/saveSaga, fetchProof, sendSettle, confirm, winners, creditOnce, computeMatchdayRoot, postRoot, mintMoments?, alert) so the resume + idempotency invariants are unit-tested with no chain or DB. Key properties: (a) PERSIST-AFTER-EACH-STEP — `runSettlement` saves the record after every advance BEFORE doing the next step's work, so a crash resumes from the last saved step and re-runs it; a step that must wait (e.g. tx not yet confirmed) returns the same step and the pass stops until a later tick. (b) ZERO DOUBLE-CREDIT — the CREDITED step calls `creditOnce(marketId, userId, qty×100)` which is idempotent per (marketId, user) (prod = the ADR-027 ProcessedEvent inbox keyed `credit:{marketId}:{userId}` + CreditLedger in one tx), so re-running CREDITED after a crash (even mid-user-list) credits each winner exactly once. (c) ONE-SHOT TX — a re-sent `settle_market` is harmless because the on-chain program is one-shot (AlreadySettled); the saga never double-settles. (d) NEVER FABRICATE — a proof 404 THROWS; `tickSettlement` catches it, ALERTS, leaves the saga parked at PROOF_FETCHED (no `result` written), and a later tick retries when the proof endpoint recovers. (e) STUCK ALERT — `checkStuck` alerts any saga not DONE whose `updatedAt` is older than 120s.
