# ADR 007 bus redis streams then kafka

Status: accepted. packages/bus abstracts publish/consume. Free tier: Redis Streams (same box as cache). Graduation trigger: >5k msg/s sustained or multi-node consumers → BUS_DRIVER=kafka (Redpanda Cloud). Topic names identical (packages/schema/topics.ts).

Notes (2026-07-07) — BUS_DRIVER flip. The flip is implemented in `createBus()` (packages/bus/src/index.ts): `BUS_DRIVER=redis` (default) → RedisBus, `BUS_DRIVER=kafka` → KafkaBus. Both drivers implement the same `Bus` interface (publish/consume/close), so services depend only on the interface and need no code change to switch. Interchangeability is guarded two ways: (1) a driver-agnostic contract suite — `packages/bus/src/contract.ts`, `busContract(name, harness)` — asserting order-per-key, at-least-once, and ack semantics; and (2) compile-time `Conforms<Driver>` proofs in that same file. Today RedisBus passes the contract against docker redis (see ADR-011 for its delivery semantics), and KafkaBus is a stub that throws on publish/consume but still typechecks against `Bus`. To graduate to Kafka: wire kafkajs into KafkaBus (key = match_id → partition, preserving per-key order) and run the SAME contract with a Kafka harness; when it is green, flip BUS_DRIVER=kafka. No consumer code changes.
