# ADR 046 The trade quote is a read-side mark-implied LMSR preview; the exact engine-q emit stays deferred (apps/api, Terminal)

Status: accepted 2026-07-10. Context: the Terminal (match-detail trading surface) needs a cost quote — SIZE → COST / AVG PX / MAX PAYOUT — for the Buy/Sell panel. Pricing a trade needs the LMSR share vector `q` and liquidity `b` (cost to buy Δ of outcome i = `buyCost(q,b,i,Δ)`, `engine/amm.ts`). ADR-042 deferred exposing `q`: it lives ONLY in the single-writer engine's journaled state and is `null` in the read model (`markets.ts` amm.q). ADR-042 named a "guarded engine-emit" as the eventual path; ADR-026 leaves the order INTAKE (and its H2/H3 ledger-atomicity + effect-outbox blockers) out of scope, so no executable fill ships yet.

Decision:
- **The quote is computed read-side from a MARK-IMPLIED q.** `q_i = b · ln(fair_i)` (recentred; the additive constant cancels in LMSR price/cost). The read model already holds `fair` (the live mark) and `bHint` (`market:{id}`), so `GET /markets/:id/quote?outcome&size&side` reconstructs q, reuses the pure `engine/amm.buyCost`, and returns `{ cost, avgPx, maxPayout, spreadMult, markImplied:true }`. `GET /markets/:id` now returns this mark-implied `amm.q` instead of `null`.
- **It is ADVISORY.** A mark-implied q equals the true engine q only when no accumulated trade skew has moved the book off the mark; once fills move q, they diverge. The quote is a preview — "server price wins at fill" (ADR-026 SLIPPAGE). It moves NO credits and touches NO engine state.
- **No engine change.** The engine is NOT modified: no new emit, no journal/effect change, so the single-writer laws and engine-guardian surface are untouched. The read layer imports only the PURE `engine/amm.ts` math (no state).
- **Display units.** The read API normalises prices to 0..100 and a winning share to 100 credits (COST/value in credits), matching the Terminal 1:1 so the fixture→API swap is direct.

Rationale: the exact-q path (the ADR-042 guarded engine→bus emit + a read cache) only earns its cost + risk when a real fill executes — and that is itself blocked on ADR-026 H2/H3. Until then a mark-implied preview is honest (advisory, server-price-wins), law-safe (read-model only), and reuses the one LMSR impl (ADR-002). It unblocks the whole Terminal quote today with zero risk to the trading core.

Consequences: `GET /markets/:id/quote` and `GET /portfolio` (positions + equity + free + live P&L, read-model only) power the Terminal. The exact engine-q emit + the order-intake gateway (with ADR-026 H2/H3) remain the follow-up that "earns the engine change" when fills go live; at that point the quote endpoint swaps its q source from mark-implied to the engine snapshot with no shape change. VERIFY: quote avgPx ≈ the mark for small size and rises with size (convexity); `markImpliedQ(fair,b)` reproduces `fair` via `price()`. Cross-ref: ADR-042 (deferred amm.q), ADR-026 (order intake + H2/H3), ADR-002 (LMSR only), ADR-027 (read projection). Reviewed by quant-reviewer (quote math).
