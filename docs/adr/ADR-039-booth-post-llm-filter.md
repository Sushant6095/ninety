# ADR 039 booth post-LLM safety filter вАФ forbidden vocab вЖТ regenerate once вЖТ drop; hard length clamp; optional env-gated TTS seam (apps/worker-jobs/src/booth-filter.ts, Phase 4)

Status: accepted 2026-07-08. Context: the AI Booth (ADR-038) narrates via a model whose output must obey booth-voice law (CLAUDE.md: NEVER bet/stake/odds/wager вАФ and now gamble вАФ in product copy). ADR-038 relied on `boothVoice` (word REPLACEMENT). This task hardens it: on a forbidden hit, regenerate ONCE, then DROP if it survives вАФ never emit a violation, even a replaced one.

Decision (implemented; 59 worker-jobs tests green):
- **`booth-filter.ts`** вАФ `FORBIDDEN` regex (whole-word, case-insensitive) over the five roots + inflections, INCLUDING `gamble/gambling/gambler` (new). `containsForbidden(s)`.
- **Mechanism in `booth.ts` `fire()`:** narrate вЖТ collapse whitespace (kills smuggled newlines) вЖТ вЙ§2 sentences (`clampSentences`) вЖТ hard `clampLength` (MAX_CHARS 240, word-boundary + ellipsis). If `containsForbidden(line)`: regenerate ONCE with `REGEN_SUFFIX` (a corrective instruction). If STILL forbidden: **DROP** вАФ log `booth.dropped` and publish nothing. Otherwise: if empty/numberless вЖТ the deterministic `fallbackLine` (template-built: clean + cites numbers); then a final idempotent `boothVoice` pass (defense in depth). This supersedes ADR-038's replace-only handling for the model path (the fallback path is inherently clean).
- **Cost note (refines ADR-038's one-call-per-trigger):** the common (clean) case is still exactly one call; a forbidden first output costs a SECOND call (the single regeneration). The 20s cooldown still bounds triggers.
- **Optional TTS seam:** `BoothTTS { speak(text, matchId) }` + `ttsEnabled(env)` (BOOTH_TTS). Injected as `BoothDeps.tts?`; when absent (default) commentary is TEXT-FIRST and never depends on audio. When present, `fire()` calls `tts.speak(line, matchId)` fire-and-forget (errors caught, never block/throw) вАФ an impl streams audio + pushes waveform state to the bubble via the match's booth channel. No provider is wired in v1; the port + env gate are the seam.

Consequences / VERIFY: (+) adversarial injection test вАФ a compromised model that echoes an injected instruction smuggling `gamble`/`odds`/`bet` + a newline via a feed-controlled `team` field yields SANE output: no forbidden vocab, no newline, cites the real move (the slot is sanitized by `sanitizeSlot`, the vocab is caught by regenerateвЖТclean). Filter units: every forbidden root+inflection flagged; allowed words + innocent substrings ("better", "betterment") not flagged; case-insensitive; `clampLength` word-boundary clamp; `ttsEnabled` off-by-default. Plus the regenerate-once-then-publish-clean and drop-on-survive integration tests. 59 tests, full gate green. (вИТ) Follow-ups: a real TTS provider impl (streaming + waveform вЖТ the WS booth channel) + a `CommentaryPayload` audio/waveform field if audio ships; a persistently-forbidden model silently drops commentary вАФ the `booth.dropped` log should feed an ops metric (like ADR-038's `booth.error`). Cross-refs: ADR-038 (the booth consumer this filters), ADR-023 (`boothVoice`, still the final idempotent pass).
