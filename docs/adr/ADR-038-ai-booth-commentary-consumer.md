# ADR 038 AI Booth commentary consumer — marks+events → triggers → one-LLM-call two-role narration → commentary.v1 (apps/worker-jobs/src/booth.ts, Phase 4)

Status: accepted 2026-07-08. Context: Phase 4 intelligence — the AI Booth must narrate market swings and match events onto `commentary.v1`, which EarlyWhistle already renders to the Telegram booth (ADR-023) and the web BoothBubble will render. booth-voice is law (CLAUDE.md: plain verbs, sentence case, NEVER bet/stake/odds/wager).

Decision (implemented; full gate green, 51 worker-jobs tests): a bus consumer `startBooth` in `apps/worker-jobs` (group `booth`) subscribing `prices.marks.v1` + `match.events.v1` and publishing `CommentaryPayload{text, voice}` to `commentary.v1`.
- **Triggers:** a price swing of |Δ| ≥ 4 points (0.04 in fair) for any outcome over a 60s window (`detectSwing` over the pruned mark history), OR a `goal` / red `card` / `penalty` / `ht` / `ft` event (yellow cards do NOT trigger). **20s cooldown per market.** Both the window and the cooldown run on EVENT time (`ts_source`, `now()` fallback), not wall-clock — so a backlog/replay burst can't fabricate swings and the cadence is replay-deterministic (aligns with the engine's time-from-events law).
- **One LLM call per trigger (cost guard):** `maybeFire` sets `lastFireAt` SYNCHRONOUSLY before the `await`, so any trigger inside the cooldown or while a call is in flight is suppressed; JS run-to-completion means no interleaving across the two consumer loops. `narrate()` is invoked exactly once per fire.
- **Two-role prompt, sanitized slots:** `buildPrompt` is a FIXED template (PLAY-BY-PLAY + MARKET COLOR + the booth-voice rule) with only two slots — ACTION and MARKET. Every feed-derived string reaching a slot is neutralized: outcome labels via `sanitizeSlot` (strip newlines/backticks/controls, clamp length); score/minute are accepted into state ONLY when numeric (a non-numeric payload field never reaches the prompt) — injection-safe.
- **Output discipline:** `clampSentences` keeps ≤2 sentences (terminator counts only before whitespace/end, so a decimal like `50.0` never splits); `boothVoice` (reused from `card.ts`) sanitizes the voice on BOTH the model and fallback paths; a real-number guarantee — if the model output is empty or cites no digit, a deterministic `fallbackLine` built from the real score/move/price is published instead.
- **LLM port + fail-open:** `BoothLLM.narrate(prompt)` is injected. Prod = `AnthropicBooth` (one Claude call via `fetch`, no SDK dep; model `claude-haiku-4-5` — the right tier for frequent lightweight narration; `max_tokens` 120 caps cost; `AbortSignal.timeout(2s)` honors the "line within 2s" target). No key → `templateBooth` (deterministic, composes from the slots). A model error/timeout is caught and the deterministic fallback line is STILL published — a trigger never yields silence, and a failure never redeliver-loops (handler resolves → acks).
- New `EventSource` value `booth` (additive, like `cortex`); FT evicts the market's state so the map stays bounded across a tournament.

Consequences / VERIFY: (+) replayed goal yields a booth line within the loop quoting the actual move (test: goal 2–1 → line contains "2-1", produced synchronously, no cadence delay); cost guard exactly-one-call-per-trigger (tests: N spaced triggers → N calls; two triggers in 20s → 1 call; event-time cooldown). Swing threshold (≥4pts fires / <4pts doesn't, quotes "50.0 → 55.0"), red-not-yellow, booth-voice sanitization, ≤2 sentences, numberless→fallback, model-error→fallback, sanitizeSlot/clampSentences/buildPrompt units — all green (51 tests). (~) The real Anthropic call is not exercised here (no key in this env); the port + template fallback are, and the fetch/timeout/parse shape is straightforward. (−) Follow-ups / known limits: (1) cooldown+swing state is PROCESS-LOCAL — keep booth single-instance; a multi-replica deployment needs Redis-shared cooldown (the `booth` consumer group would otherwise split state). (2) `hasNumber` is a syntactic check — it does not verify the model's number matches the data (the fallback path is the guaranteed-correct one); a stricter number-match verifier is possible if hallucination proves an issue. (3) the web leg — an `api` bridge consuming `commentary.v1` → the WS `booth` channel, and a designed BoothBubble (currently a stub, "build against design/screens crop") — is a separate UI task; the Telegram booth consumes `commentary.v1` today. (4) `settlementProof`/other consumers likewise don't `safeParseEvent`; adopting runtime envelope validation at consumer boundaries is a systemic follow-up. Cross-refs: ADR-023 (EarlyWhistle + the `boothVoice` filter reused here), ADR-022 (the marks this consumes), ADR-020 (the bus planes).
