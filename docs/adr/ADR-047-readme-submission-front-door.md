# ADR 047 README rewritten as the submission-grade front door — every claim traceable to a file or a test, or cut

Status: accepted 2026-07-13. Context: the TxODDS World Cup Hackathon (track: Prediction Markets & Settlement; India Buildathon deadline 2026-07-13, global track to 2026-07-19) is judged from the repo's front door. The old README under-sold the system and mixed in stale "OMNIPITCH" copy. This is a documentation/positioning decision, not a code-architecture change — no service, engine, or bus behaviour moves.

Decision:
- **README.md is the front door.** Ten sections: hero (home/terminal/northstar screenshots + badges) · what-it-is · the 5-beat demo loop (goal→halt→reprice→decaying spread→AI Booth narrates, per ADR-005/038/039) · TxLINE-as-primary-data-source endpoint table (A1–A3/F1/S1–S4/O1–O3/K1 each mapped to the surface it powers, from `docs/SUBMISSION.md`) · architecture (L0–L6 layer map + a 5-line data-flow contract + a mermaid diagram TxLINE→ingest→bus→cortex→engine→api→web with the Solana settle branch) · the settlement story · built-vs-next · a 10-minute run guide · the full 46-ADR index (all links verified) · team/license/play-money disclaimer.
- **Brand copy is Ninety, namespace stays `@omnipitch/*`** (ADR-044): zero "OMNIPITCH" in user-facing README prose; the internal package namespace is left AS-IS as a namespace, not the brand.
- **Play-money law upheld** (CLAUDE.md copy law): no bet/stake/odds/wager in copy; "odds" appears ONLY as literal TxLINE endpoint/feed names.
- **The settlement story is honest and addressed to the sponsor:** `validateStatV2` does not bind finality on-chain and stat leaves carry no Action field, so a mid-match Seq can be selected to forge a settle — so we fail-closed (`SETTLEMENT_LIVE=false`) rather than ship a forgeable settle (ADR-036/037).
- **Honesty stance (judges punish overclaiming):** README states plainly that the frontend is fixture-wired today (`apps/web/src/lib/api.ts` has 0 importers; `apps/web/src/lib/fixtures.ts` is imported by 23 files), `mint_moment` is a stub (ADR-032/041), and on-chain settlement is fail-closed.
- **Metrics are counted, not estimated:** 257 automated tests pass = 248 TypeScript (vitest: api 144, worker-jobs 68, worker-ingest 15, schema 10, bus 6, chain 5) + 9 Python (pytest, worker-cortex); plus 5/5 Anchor tests (ts-mocha, localnet). No LICENSE file is committed — the README says so rather than inventing one.

Consequences: the repo now presents itself accurately to judges — nothing in the README overstates what runs. Principle established and binding on future README/SUBMISSION edits: **every claim is traceable to a file or a test; unverifiable claims are cut.** VERIFY: no "OMNIPITCH" and no bet/stake/odds/wager (outside TxLINE endpoint names) in `README.md`; test counts match the runners; every ADR link in the index resolves.
