# ADR 052 GSAP added as a second sanctioned animation library alongside Framer Motion (reverses the "Framer-only" clause of ADR-050/ui-craft §7 for heavy choreography)

Status: accepted 2026-07-13. Supersedes: only the "Framer Motion is the ONLY animation library" clause of `ui-craft` §7 (and the earlier in-session "keep GSAP out" decision). Everything else about the motion law is unchanged.

Context: `ui-craft` §7 named Framer Motion the sole animation lib and banned any second one — the same clause that (correctly) kept recharts out. Framer is excellent for micro-interactions but is not the right tool for heavy, timeline-driven choreography — SVG stroke drawing, scroll-pinned scrollytelling, per-character text reveals. GSAP is the standard for those, went fully free/open-source in 2025 (all former Club plugins now ship in the npm package), and the repo owner elected to use it heavily. This ADR reverses the ban for GSAP specifically, with a clear split so the two libs don't overlap.

Decision:
- **GSAP is a sanctioned animation library alongside Framer Motion.** Installed as `gsap@^3.15` + `@gsap/react@^2.1` (regular deps in `apps/web`) — NOT the vendored `~/Downloads/gsap-public` folder (npm ships all plugins now). No third animation lib beyond these two.
- **The split (the whole point):** **Framer Motion** owns micro-interactions — price flash, number rolls, badge pulse, reveals, sheet/tooltip/dialog enters. **GSAP** owns heavy choreography — the Momentum River stroke draw (DrawSVGPlugin), how-it-works scrollytelling (ScrollTrigger/ScrollSmoother), hero text (SplitText), and complex timelines. Never animate one element with both.
- **One motion law still governs both.** Import gsap + useGSAP from **`apps/web/src/lib/gsap.ts`**, which registers plugins once and pins `gsap.defaults` to a `CustomEase` ("ninety") mirroring `design/motion.ts`'s `easeOut` cubic-bezier and its 200 ms transition. GSAP tweens must animate transform/opacity only, honor `prefers-reduced-motion` (via `useGSAP` + `gsap.matchMedia`), and avoid idle/infinite motion — the same rules Framer already follows.
- **Law files amended:** `ui-craft` §7 (two-lib split + the banned-list carve-out), `design-cop` criterion 5 MOTION (both libs sanctioned; timings still trace to the tokens), and `CLAUDE.md` design law. Heavier plugins (DrawSVG/SplitText/ScrollSmoother) are imported+registered in the single surface that needs them to keep the shared bundle lean.

Consequences: design-cop no longer fails GSAP usage that respects the motion tokens. Risk — two animation systems can drift or double-animate; bounded by the explicit split, the shared `lib/gsap.ts` defaults, and the "never both on one element" rule. VERIFY: `pnpm --filter web build` green with GSAP wired; `gsap`/`@gsap/react` in `apps/web` deps; `lib/gsap.ts` registers ScrollTrigger + CustomEase and pins the token ease; the three law files updated. Cross-ref: ADR-050 (the "primitives-default"/consistency posture this refines), ADR-045 (inline-SVG viz — the River stroke GSAP will draw), ADR-049 (design-cop criteria), the `momentum-river` skill (the River hero), and the 8 GSAP skills (`gsap-core`/`react`/`scrolltrigger`/`plugins`/`timeline`/`frameworks`/`performance`/`utils`).
