# ADR 014 txline wire schemas live in packages/txline not packages/schema

Status: accepted 2026-07-07. Context: ADR-010 established that internal bus event shapes live in `packages/schema` and no service ever defines its own event shape again. This session added typed wrappers for the TxLINE REST/SSE endpoints (scores snapshot/updates/stream/stat-validation, fixtures, odds snapshot/updates/stream) and needed a home for the zod schemas describing TxLINE's *wire responses* — putting those in `packages/schema` would blur two different contracts. Decision: (1) TxLINE wire-response schemas (zod) live in `packages/txline` (`src/scores.ts`, `src/odds.ts`, `src/fixtures.ts`, `src/statkeys.ts`), owned by the only package allowed to call TxLINE; `packages/schema` stays the home of the *internal* bus Envelope/AnyEvent contract only. The boundary: external API DTOs are txline's concern; worker-ingest maps those DTOs onto the canonical Envelope (`packages/schema`) — that mapping, not shared schema types, is where the two contracts meet. This does not violate ADR-010: those are bus events, these are external wire types. (2) Every wrapper on `TxLineClient` returns a parsed schema type (`z.infer`), never `any` — explicit return-type annotations plus `schema.parse()` inside a private `getParsed()`/`stream()` helper make tsc enforce it. (3) The SSE endpoints (S3 scores stream, O3 odds StablePrice stream) are exposed as async generators that parse each frame's data payload with the schema, yield typed events, and fire an `onGap` callback on a seq gap (recovery = the consumer re-fetching a snapshot, per the txline-integration skill). (4) The four previously-⚠ paths (O1 `/api/odds/snapshot`, O2 `/api/odds/updates`, O3 `/api/odds/stream`, F1 `/api/scores/schedule`) are filled by inference from the confirmed scores endpoints' shape and marked ⚠ inferred in TXLINE-MAP §1 pending Day-0 confirmation; `statkeys.ts` carries the two confirmed keys (1002 home goals, 1003 away goals) with the full table deferred to Day-0. Consequences: (+) clean layering — external contract in txline, internal contract in schema, mapping in ingest; consumers of the wrappers get real types, not `any`. (+) verification runs with no live API: a deterministic mock (real WC26 data) drives all 8 wrappers, and the verify script asserts parsed shapes + dual headers + cache + refresh + network guard and (with `--emit`) writes one sample per endpoint to `docs/txline-samples/`, doubling as `pnpm --filter @omnipitch/txline test`. (−) three odds paths + the fixtures path + the full statKey table remain ⚠ inferred/partial until Day-0 confirmation against the live docs. No change to any CLAUDE.md law or hook. Relates to ADR-010 (bus event contract), ADR-013 (txline client).
