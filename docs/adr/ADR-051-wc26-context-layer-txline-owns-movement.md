# ADR 051 The WC26 context layer: a two-source split where TxLINE owns everything that MOVES and worldcup26 owns everything that SITS STILL

Status: accepted 2026-07-13. Context: Ninety must feel like a real WC26 football product — flags, crests, groups, standings, the 104-match bracket, stadiums, the fixture calendar — without ever weakening TxLINE as the PRIMARY source. The bounty requires TxLINE as primary and disqualifies bolt-on second feeds, and markets settle against TxLINE proofs on-chain (ADR-035/036/037), so pricing and match state must never originate anywhere else. We add a second data source for STATIC context ONLY, structurally fenced so it cannot leak into anything live. Only `apps/web` was touched — `apps/api`, `packages/`, `programs/` are untouched (architecture law upheld).

Decision:
- **The hard rule: TxLINE owns everything that MOVES; worldcup26 owns everything that SITS STILL.** TxLINE (untouchable, primary) = live scores, goals, the halt trigger, odds/marks, prices, match state, settlement proofs. worldcup26 (new, static context) = flags, crests/team names + metadata, stadiums, the 12-group draw + standings, the 104-match bracket skeleton with placeholder labels ("Winner Group A", "Winner Match 101"), fixture calendar. **Tie-breaker, verbatim: "if a field moves during a match → TxLINE."**
- **Baked at build time, never called at runtime.** `apps/web/scripts/fetch-wc26.mjs` fetches `/get/{teams,groups,stadiums,games}` (source: worldcup26 — github.com/rezarahiminia/worldcup2026, worldcup26.ir, ISC; read endpoints are PUBLIC, no API key / account needed), validates HARD (exact counts 48/12/16/104 + required keys via hand-rolled assertions, no new zod dep per the no-new-libraries guardrail), and writes atomically (all four files or none) to `apps/web/src/data/wc26/*.json`. Typed loaders in `index.ts` + `types.ts`; `pnpm --filter web wc26:refresh` re-bakes. The build reads local JSON only — if worldcup26.ir 404s during judging the site does not notice.
- **The hard rule is enforced STRUCTURALLY at bake time, not by convention.** Every live field is stripped (`home_score`/`away_score`/`home_scorers`/`away_scorers`/`finished`/`time_elapsed`), and knockout games' resolved team refs are NULLED (only static placeholder labels survive) — because who advances depends on real results = TxLINE/reality, not worldcup26's own simulation. Group-stage games keep their fixed-draw team refs. This makes it structurally impossible to render a worldcup26 score or knockout result.
- **Flags stay on flagcdn at runtime** (a distinct, reliable CDN already allowed in `next.config`), not downloaded/optimized to `public/` — the runtime-dependency risk this ADR targets is worldcup26.ir (baked away), not flagcdn, and self-hosting would have forced a `sharp` image dep against the no-new-libraries guardrail. worldcup26 ships national FLAGS, not separate crest logos (for a national team the flag IS the identity); missing flags fall back to 2-letter initials and never break a row.
- **First surface: `/competition`** — all 12 group tables (MP/W/D/L/GF/GA/GD/Pts), HD flags, team names + FIFA codes. Routes split: `routes.competition` now = `/competition` (was aliased to `/bracket`), `routes.bracket` added = `/bracket`, both in the nav. The top-two qualification zone is highlighted with a **NEUTRAL tint, not green/pink** — semantic-colour law: up-green/down-pink stay price-only. Attribution: worldcup26 (ISC) credited in the README (Credits + run-guide) and an in-product Footer credits line.

Consequences: Ninety gains a football-product context layer with zero risk to the primary source — no second feed can price, settle, or move match state, by construction of the bake step, not by discipline. VERIFY: `fetch-wc26.mjs` asserts 48/12/16/104 and writes atomically; baked `games.json` has no live-score keys and knockout `home_team`/`away_team` refs are null while placeholder labels remain; `pnpm --filter web wc26:refresh` re-bakes; grep confirms no runtime fetch of worldcup26.ir; `/competition` renders 12 group tables with the qualification zone in a neutral (non-green/pink) tint; README + Footer carry the ISC credit.
