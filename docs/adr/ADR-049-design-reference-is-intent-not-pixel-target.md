# ADR 049 The design reference is INTENT, not a pixel target — supersedes ADR-043's frozen-ceiling reading, unblocking the consumer/fan layer

Status: accepted 2026-07-13. Supersedes: only the "reference = immutable pixel ceiling" reading of ADR-043. ADR-043 is NOT edited — it stays as the historical STEP-0 Home lock. (Numbered 049, not 047 as the originating prompt guessed: ADR-047 and ADR-048 were already taken by the README front-door and the Vercel deploy.)

Context: for ~5 days the ui-craft loop could not improve the UI *by construction*. `design-cop` criterion 9 (FIDELITY) failed any structural deviation from `design/screens/home.png`, so every genuine improvement scored as "drift" and was reverted — the reference behaved as a CEILING, not a floor. ADR-043 froze that reference and ADRs are immutable-by-policy. Meanwhile CLAUDE.md's "more libraries = more slop" plus design-cop #2 (tokens-only) and #9 (fidelity) meant any imported primitive tripped the rubric on arrival, so the agent rationally never reached for one. We are now building the **Consumer & Fan Experiences** layer, which REQUIRES structural change. The system must be fixed before the screens, or the screens revert.

Decision:
- **The design reference (`design/screens/*.png`) is INTENT, not a pixel target.** Structure MAY improve beyond it. A screen that *beats* the reference is passing, not drifting. The PNG is a floor and a north star, never a ceiling.
- **What stays absolutely fixed (still hard law, unchanged):** the tokens (`apps/web/src/design/tokens.ts` via CSS vars — zero raw hex); the **play-money copy law** (never bet / stake / odds / wager / gamble — say price, trade, credits); the **architecture law** (web reads REST/WS only, never DB or chain writes; TxLINE only in `packages/txline`, Solana tx only in `packages/chain`); and **River-as-hero** on the live match view (≥45% viewport, lightweight-charts, `series.update()`).
- **`design-cop` criterion 9 (FIDELITY) is retired and replaced** by three criteria — 9 CONSISTENCY (one shell across every page), 10 ELEVATION (beat the reference, don't merely match), 11 FEELING (a fan who does not trade would want to look). The rubric shifts from "match the crop" to "one consistent shell that elevates and delights inside the fixed laws." Criteria 1–8 are unchanged.
- **`ui-craft` §7 is amended:** third-party primitives (dialog, tabs, tooltip, sheet, command, scroll-area) are the DEFAULT for generic UI — pull them in, then re-skin to tokens; the token law governs the OUTPUT, not the moment of import. Ninety-specific pieces (River, MatchCard, PriceChip, trade ticket, ProofBadge, Booth) stay hand-built.

Consequences: the loop can now raise the bar instead of reverting to it. Risk — "structure may improve" could re-admit slop; it is bounded by the unchanged tokens/copy/architecture/River laws and by the new ELEVATION + FEELING criteria, which demand intentional delight rather than additive noise. VERIFY: `design-cop.md` criteria 9–11 updated and 1–8 verbatim; `ui-craft/SKILL.md` §7 makes primitives the default with the hand-built exception list.
