# ADR 065 Credibility fixes — /match/[id] renders the correct market, and /proofs is honestly fail-closed (not fabricated)

Status: accepted 2026-07-16 (`merge/live-integration`, `352490e`). Cross-ref: ADR-036/037 (the settlement forge finding + fail-closed decision this surfaces honestly), ADR-051 (two-source law — TxLINE owns the RESULT, chain holds the PROOF), ADR-049 (feeling/design-cop).

Context: two credibility bugs, both confirmed on the live deploy, both zero-credential.
1. **`/match/[id]` rendered the same match (Australia v Egypt) for every id.** Root cause: `app/match/[id]/page.tsx` did `await params` but never destructured `id`, and `TerminalScreen`/`MatchColumn` were hardwired to the featured market — so `/match/bra-kor` and `/match/eng-sui` both showed the featured fixture. On a real product this reads as "nothing works."
2. **`/proofs` shipped 8 fabricated Solscan signatures** (`"5Kx7bQ…9f2aTx"` — 13 chars with a literal ellipsis baked into the data) under the words "anyone can verify it." Every link 404s on Solscan. On a Solana track this reads as dishonesty, not incompleteness — the single worst item on a public repo.

Decision:
- **`/match/[id]` — parameterize by id, fail closed on a miss.** Destructure `const { id } = await params`; look it up via `marketByMatchId(id)` in `lib/fixtures`; unknown id → Next 15 `notFound()` (real not-found UI, **never** a silent fallback to the featured match). `TerminalScreen(matchId)` → `MatchColumn(matchId)` dispatches by id: the **featured id (`wc26-aus-egy`) renders `FeaturedMatchColumn` byte-unchanged** so the halt money-shot is preserved intact; every other id renders a new `PlainMatchColumn` seeded from `useMatchLive(id)` + the fixture record (header/score/teams/River/price cells, PRE/LIVE/SETTLED states, a derived-LMSR trade panel, and honest depth tabs — real baked squads via `SquadPitch`, else an honest "pending" pitch). AUS-EGY-specific rails and the "Replay the halt" dock tool are gated to the featured market. All in-app links route via `routes.match(id)` to the correct store keys.
- **`/proofs` — Path A, relabel honestly (fabrication is never acceptable).** `ProofBadge` (the root fix) emits a Solscan `<a>` **only** for a real 87–88-char base58 signature; otherwise it renders a non-link "proof pending" chip — killing every dead Solscan href app-wide in one place. `lib/proofs.ts` fabricated sigs (and fake slots) removed; the page is reframed to the truth: settlement is **fail-closed on purpose** — `validate_stat_v2` does not bind finality on-chain (ADR-036/037), so settlement is disabled rather than shipped, and proofs go live once TxODDS confirms finality binding. The page links the ADRs + `/how-it-works`, not a 404. Sibling fabrications swept (`moments.ts` `mintSig: null`, `ProofFlow`, a dead `LeftRail` Solscan link). **Path B rejected:** `SETTLEMENT_LIVE = false` means there are no real settle transactions to link; a real sig is 87–88 chars and none exist.

Consequences / VERIFY: `tsc` clean; **axe 0** on `/match/wc26-{aus-egy,bra-kor,eng-sui}` + `/proofs`; **design-cop PASS** (match-live/pre/proofs; the one CONCERN is a pre-existing ~1280px TradePanel collision inside the byte-unchanged featured code, correctly left untouched). Read-out-loud proof: bra-kor = Brazil 2–0 South Korea 55′ (BRA 86.7/9.6/3.7), eng-sui = England v Switzerland PRE (52/26/22), aus-egy = the money-shot — three different matches. `grep` proves no ellipsis in any sig and no dead Solscan hrefs. The `/proofs` ADR link resolves (301 → the `docs/adr` folder). Note: `notFound()` renders Next's not-found UI but HTTP 200 (RSC streaming) — the credibility defect (silent featured-match fallback) is gone, which is the point. OPEN: the mandatory **live-deploy** verification is blocked on the Vercel Deployment-Protection toggle (the preview is SSO-gated); worktree verification stands until it's flipped.
