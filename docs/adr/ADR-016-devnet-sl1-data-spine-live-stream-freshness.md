# ADR 016 devnet SL1 data spine verified live — stream freshness + keepalive handling (06 GREEN)

Status: accepted 2026-07-07. Context: ADR-015 turned prompt 05 VERIFY GREEN (auth + snapshot endpoints world-verified on devnet), but the SSE data spine (scores S3 / odds O3 streams) had returned no events in that session's windows, so ADR-015 could only document a stream event = one inferred `ScoreState`/`OddsTick` rather than an observed one, and prompt 06 needed live in-play fixtures to actually measure stream freshness before 07 (the proof-spike) could proceed. Session 06-LIVE ran the live runner (now at `scripts/txline-live.mjs`, the composition layer; the repo root gained a `@solana/web3.js` devDep so a root-level ESM script resolves web3) against txline-dev + devnet while WC26 Round-of-16 fixtures were in play, to capture SSE samples, measure freshness (arrival vs match clock), and gate 07 on the result. Decision / findings (verified live 2026-07-07): (1) In-play confirmed — `/api/fixtures/snapshot` showed 6 WC fixtures at `GameState=1` (USA v Belgium, Argentina v Egypt, Switzerland v Colombia, France v Morocco, Norway v England); the odds SSE stream is very active (~90 real `OddsTick`/min) while the scores SSE stream is sparse and emits mostly bare `{Ts}` keepalive frames between real events. (2) Freshness: devnet SL1 stream TRANSPORT latency (feed-emit → client) ≈ 0.7–1.2s median across both streams (arrival − `event.Ts`, normalizing scores `Ts` in seconds vs odds `Ts` in ms) — sub-second/fresh, NOT materially delayed, so NO topology STOP; the SL1 tier's documented 60s delay is data-vs-real-match staleness by design, not transport, so true real-time trading still wants mainnet SL12 but devnet SL1 is fine for dev / replay / the 07 proof-spike. (3) Keepalive handling: the SSE stream wrapper (`packages/txline/src/client.ts`) now `safeParse`s each data frame and SKIPS anything that isn't a valid event (e.g. the `{Ts}` keepalives) instead of throwing — never quote blind; the mock (`packages/txline/src/mock.ts`) now emits a `{Ts}` keepalive followed by a real `ScoreState` so the CI test proves the skip. (4) Samples: a real `OddsTick` from the live odds stream is captured at `docs/txline-samples/odds-stream-event.json`; a real scores-stream `ScoreState` did NOT occur in the capture windows (only keepalives), so no scores-stream-event sample is committed — a scores stream event shares the `ScoreState` shape of `scores-snapshot.json`, and the keepalive-only capture was removed rather than commit something misleading; the fixtures-snapshot sample now includes World Cup fixtures. (5) 06 is now VERIFY GREEN (world-verified), unblocking 07; the 07 spec (`.claude/prompts/1-data-spine/07-proof-spike.md`) gains a STEP 0 to close ADR-015's open ⚠ (statKey catalog + FINAL-goals key + FINISHED-field discovery), and the VERIFY three-states doc gains a WORLD-GATED rule (07–12, 19–24 gated on NOW.md not mentioning txline-env). Consequences: (+) the live data spine is world-verified end to end (auth + all endpoints + fresh sub-second streams), so 07 can proceed on devnet. (−) a real scores-stream event shape is confirmed only by inference from snapshot/updates (`ScoreState`) — the stream was empirically observed emitting only keepalives during the windows; (−) precise data-vs-real-match latency (the 60s tier delay) is taken from the docs, not independently measured. No CLAUDE.md law or hook changed — the "only chain builds Solana txs" and "only txline calls TxLINE" laws were upheld. Extends/cross-refs ADR-015 (live corrections), ADR-013 (client), ADR-014 (wire schemas).
