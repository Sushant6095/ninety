# ADR 032 on-chain Moments (Metaplex Bubblegum cNFT) CUT for v1 — feature-flagged MOMENTS_ONCHAIN off; off-chain moment metadata is the v1 path (prompt 21)

Status: accepted 2026-07-08. Context: prompt 21 — mint a market's biggest-swing "Moment" as a Metaplex Bubblegum compressed NFT via CPI to a prepared tree, behind a `MOMENTS_ONCHAIN` feature flag. The prompt itself carries an explicit escape: "If Bubblegum friction > half a day → stop, flag off, /adr the cut. VERIFY: one cNFT minted on devnet visible via Helius DAS; OR documented cut." Decision: TAKE THE DOCUMENTED CUT — MOMENTS_ONCHAIN defaults OFF; Moments live off-chain in v1; the on-chain cNFT mint is deferred. This is evidence-based, not an assumption: (1) the mpl-bubblegum Rust CPI is NOT the blocker — `mpl-bubblegum = "1.4.0"` resolves and `cargo check` COMPILES cleanly against this program's deliberately-old, rustc-1.79-pinned dependency graph (ADR-030) with only 3 small added crates, so a future `mpl_bubblegum::MintV1Cpi` wiring is viable. (2) The BLOCKER is the end-to-end VERIFY, which needs external setup that is not provisioned and exceeds the half-day threshold for a cosmetic feature: (a) a Helius API key — `HELIUS_API_KEY` is unset (empty in `.env.example`), so the "visible via Helius DAS" check cannot run at all; (b) a pre-created devnet Bubblegum concurrent-merkle tree (`create_tree` + funding), which does not exist; (c) the upstream Moment pipeline (render the river-cliff PNG → upload) is still a stub (ADR-023). (3) Moments are cosmetic (a shareable NFT of a price swing), not part of the play-money trading/settlement core. Deliverables (the cut, done): `MOMENTS_ONCHAIN` feature flag (env, off by default) + an off-chain `recordMoment()` seam in `apps/worker-jobs/src/moments.ts` — when off it returns `{ onchain: false }` (v1: the Moment row + imageUri only); when on it fails loudly (minting not yet wired) rather than silently no-op'ing. The on-chain `mint_moment` instruction (programs/omnipitch_core) stays a documented no-op stub gated behind the same flag. Consequences / VERIFY (documented cut — the prompt's sanctioned alternative to a devnet mint): worker-jobs unit test green (MOMENTS_ONCHAIN defaults off; `recordMoment` takes the off-chain path); the mpl-bubblegum Rust CPI compiles (feasibility proven by `cargo check`); the devnet mint + Helius DAS confirmation is the explicitly-deferred portion. (~) No new runtime dependency is shipped — mpl-bubblegum was added only for the compile probe and REVERTED (keeping the build lean); it is re-added when the on-chain path is wired. Play-money invariant unaffected (a cNFT Moment is a cosmetic collectible, never a payout). (−) To lift the cut later: provision a devnet Bubblegum tree + a `HELIUS_API_KEY`, wire `mint_moment` to `MintV1Cpi` (tree_authority / leaf_owner / merkle_tree / log_wrapper / compression_program / bubblegum_program accounts), build the Moment PNG+upload pipeline, flip `MOMENTS_ONCHAIN` on, and verify the asset via Helius DAS (`getAsset`). The `mint_moment` tx builder is tracked under prompt 23 (packages/chain). Cross-refs: ADR-023 (Moments/EarlyWhistle surface), ADR-030 (the anchor build unblock that let the CPI compile-probe run), prompt 23 (mint_moment tx builder + Helius webhook types).
