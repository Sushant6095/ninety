# ADR 023 EarlyWhistle Telegram live match cards — pure renderCard, global scheduler, per-match state machine (Sessions 52+52B fused)

FIRST 7-surfaces-phase ADR. Consumes the data spine (`match.events`, `prices.marks`, `commentary`, `settlement` on the bus) and the moments renderer; builds on ADR-020 (bus planes / inter-service law), ADR-018 (ingest → `match.events`), ADR-022 (`prices.marks`), ADR-017 (settlement proof). Status: accepted 2026-07-07. Context: `apps/worker-jobs` renders EarlyWhistle's Telegram live match cards — an auto-editing live-channel card per fixture and a `/match` one-shot — off the bus. DEPS-GATE OVERRIDE (recorded prominently): 52B's prompt gated on "Session 52 GREEN in the ledger"; Session 52 (the EarlyWhistle alert wire) did NOT exist — no ledger row, no `earlywhistle.ts`, no 7-surfaces prompt dir. Per the gate I STOPPED and reported; the human EXPLICITLY chose to OVERRIDE and build 52 + 52B fused from scratch in one session. Consequence: there is NO separately-verified Session-52 alert-wire baseline underneath 52B. Decision (implemented, gate-green, adversarially reviewed 2026-07-07; test runner switched to vitest): (1) CARD = a PURE `renderCard(state)` (`card.ts`) shared by the live card and `/match` — golden-snapshot tested. MarkdownV2: price rows render inside a ``` ``` ``` code fence for monospace/tabular alignment (design law); everything else via `escapeMd`. The ≤3500 hard cap truncates the RAW inputs and drops optional lines, NEVER slices the final string (slicing would cut a MarkdownV2 escape or fence in half → 400). A booth-voice copy filter enforces the CLAUDE.md copy law on outbound dynamic strings (never bet/stake/odds/wager → trade/credits/price). (2) SPARKLINE = a bounded per-market ring buffer (`matchcard.ts`, 15-min window + 512-point cap → no unbounded memory), rendered as a 16-bucket unicode sparkline; FROZEN (snapshot) while HALTED, cleared on reopen AND on settle. (3) GLOBAL SCHEDULER (`scheduler.ts`): ONE 25/s token bucket across ALL matches (one bot = one flood-control budget); events strictly beat edits; edits coalesced per message (only the latest render matters); per-message edit throttle ≥4s; 429 backs off `retry_after` (or a default when omitted) and re-queues; an EVENT is NEVER dropped (bounded transient-error retry); a re-entrancy guard prevents overlapping interval-driven pumps. (4) STATE MACHINE per `match_id` (independent cards): kickoff → post + PIN; cadence edit every 8s if a mark changed, immediate on goal/red/halt/settled, never faster than 1 edit/4s per message; HALTED flips the header to "repricing" + freezes the sparkline; SETTLED does a final edit (result + "proof verified on Solana" + Solscan link) → unpin → post the Moment PNG as a photo reply. The card slot is reserved SYNCHRONOUSLY (no double-card race); posting is idempotent on `messageId` (at-least-once redelivery safe); a settled event arriving before any card still runs the FULL settle sequence. Bus handlers are fire-and-forget (never block the pipeline); a Δms latency log is kept; settled cards are evicted after a grace period (bounded memory). Telegram access is behind an injectable `TelegramClient` (thin fetch impl + mock) — NO heavy dependency added. Consequences / VERIFY: (+) gate-green; the pure `renderCard` is golden-snapshot tested and shared by both surfaces. (+) ADVERSARIAL PASS (4-lens / 15-agent) found 9 confirmed defects before ship: 1 CRITICAL — a literal `+` on `topSwing` was unescaped and `+` is MarkdownV2-reserved → EVERY card would 400, un-retried, feature emits nothing, AND the happy-path golden snapshot had LOCKED the broken output (worth recording: the golden test blessed the critical bug; the adversarial pass is what caught it); 3 HIGH — pump re-entrancy double-editing within the 4s throttle, events permanently dropped on any non-429 error, and a check-then-await-then-set race double-posting/pinning one match; plus medium/low — kickoff redelivery re-post, settled-first bypassing the Moment photo, unbounded cards/lbCache, halt→settled stale sparkline, first-render stale 0/0 leaderboard. ALL 9 fixed with regression tests. (−) DEPS-GATE OVERRIDE: no separately-verified Session-52 alert-wire baseline exists under 52B. (~) `main.ts` wires `getFixture`/`getLeaderboard`/`getMomentPng` as STUBS — the Redis-backed readers (`fixtures:current` hash, `lb:global` zset, moments renderer output) are PENDING; the `/match` handler is unit-tested but the `getUpdates`/webhook receive loop is NOT wired. Play-money invariant unaffected; NO admin/result path added; no CLAUDE.md law or hook changed (the booth-voice filter ENFORCES the existing copy law, it does not alter it). Cross-refs ADR-020 (bus planes), ADR-018 (`match.events`), ADR-022 (`prices.marks`), ADR-017 (settlement proof / Solscan).
