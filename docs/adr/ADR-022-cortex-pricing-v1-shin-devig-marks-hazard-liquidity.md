# ADR 022 cortex pricing v1 — Shin de-vig marks + hazard-driven liquidity

Builds on ADR-004 (python cortex), ADR-015 (feed reality), ADR-018 (ingest → `odds.raw`); depends on ADR-020 (bus planes / inter-service law). Status: accepted 2026-07-07. Context: `apps/worker-cortex` (Python) is the first pricing consumer on the bus — it subscribes `odds.raw` (ADR-018), de-vigs each tick into a fair Mark summing to 1, publishes canonical Envelopes on `prices.marks.v1`, and emits a hazard hint the engine uses to set dynamic liquidity `b(t)`. ADR-004 reserved pricing math for Python (penaltyblog); this ADR is the v1 realization, and — like ADR-015/018 — the live feed corrected the plan. Decision (implemented, gate-green, quant-reviewer 6/6 PASS 2026-07-07): (1) DE-VIG = Shin via penaltyblog with a POWER fallback (`devig.py`). KEY REALITY FINDING: TxLINE ships a demargined book (`TXLineStablePriceDemargined`) whose booksum ≈ 1 and is often slightly UNDER 1; penaltyblog's Shin root-finder (scipy `ridder` on z∈[0,100]) has NO sign change on an under-round book and RAISES — so the LIVE path uses the power fallback, while Shin is still exercised on genuinely margined books. The power fallback is LOAD-BEARING, not decorative. (2) BLEND `fair = 0.2·model + 0.8·consensus` (quant canon; `pricing.py`). The model term is 0 in v1, so the blend COLLAPSES to pure devigged consensus — still sums to 1, NOT `0.8·consensus`. The 0.2 model term (Dixon-Coles intensities + ~500-particle filter) is DEFERRED — the "stretch only if ahead" item; `dixon_coles.model_probs` returns `None`. (3) MARKS carry the MARKET's own outcomes summing to 1. Because the live TxLINE feed is per-market (OVER/UNDER and ASIAN HANDICAP — 2-outcome), NOT clean 1X2 (ADR-015), a mark's fair is keyed by the market's outcomes (over/under, part1/part2), not H/D/A; `market_id` disambiguates. This forced a SCHEMA change: `packages/schema/src/market.ts` `Mark.fair` was generalized from `z.record(Outcome=H|D|A, number)` to `z.record(z.string(), number)` — still accepts H/D/A once a model derives 1X2. The 1X2 derivation awaits the deferred model. (4) PROVENANCE: added `"cortex"` to the `EventSource` enum (`packages/schema/src/envelope.ts`). Marks are published as canonical Envelopes `{source:"cortex", type:"mark", payload:Mark}` on `prices.marks.v1`, one mark per odds tick (`source_seq` inherited from the tick → idempotent on `(cortex, source_seq)`). (5) HAZARD v1 heuristic (`hazard.py`): recent-tick variance (leading-outcome prob) + goal spike (exponential decay, τ=90s) + late-game scaling (ramps up after minute 75); `b_hint = b0·(1+κ·hazard)` clamped `[b0, 3·b0]` (b0=300, κ=1). This is the dynamic-liquidity signal the engine consumes to set `b(t)`. Calibration constants (`VAR_SCALE`, `GOAL_SPIKE`, τ, κ) are tunable knobs. 20ms/tick budget met. (6) `bus.py` is the PYTHON ARM of the bus: it speaks the SAME Redis Streams wire format as `packages/bus` (`XADD` with fields `{key, data:<json envelope>}`, consumer groups for at-least-once, identical topic names). GOVERNANCE POINT: ADR-020's law "raw Redis/Kafka forbidden outside `packages/bus`" must be read as "the bus CONTRACT" — TS impl `packages/bus`, Python impl `apps/worker-cortex/src/cortex/bus.py` speaking the identical wire format + topic names. `cortex/bus.py` uses raw redis but IS the bus (its Python implementation of the same contract), NOT an ad-hoc side channel and NOT a second plane; this is the sanctioned way a Python service joins the bus. FLAGGED (not applied): a future CLAUDE.md clarification line could make this explicit for hook enforcement — that needs human sign-off and is NOT changed here. (7) TOOLING: `pyproject` requires Python ≥3.11; a python3.11 venv (`apps/worker-cortex/.venv`, gitignored) holds numpy/redis/penaltyblog 1.11.0/pytest. The `package.json` `test` script runs pytest via the venv with a graceful skip when `.venv` is absent, so the turbo gate stays green on machines without the venv. Consequences / VERIFY: (+) every replayed mark's fair sums to 1±1e-6 (exact 1.0 on the real samples); Shin matches an independent hand-computed bisection reference on a margined book; a replayed goal spikes the mark's hazard then it decays; quant-reviewer 6/6 PASS. (+) cortex is the FIRST Python consumer on the bus, proving the contract crosses languages without a second plane. (~) v1 fair = pure devigged consensus; the 0.2 Dixon-Coles model term is deferred (stretch only if ahead). (−) marks are per-market (over/under, handicap), not 1X2 — 1X2 derivation awaits the deferred model, continuing the ADR-015/018 "reality wins" thread. Known follow-ups: (a) the deferred Dixon-Coles model term; (b) the optional CLAUDE.md clarification for the Python bus arm (human sign-off); (c) a penaltyblog-API import smoke check (already exercised by the passing tests). Play-money invariant unaffected; NO admin/result path added.
