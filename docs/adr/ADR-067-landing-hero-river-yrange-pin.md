# ADR 067 Landing hero River draws big — pin the price axis (fix the hero void)

Status: accepted 2026-07-16 (`merge/live-integration`, `8eb7eb7`). Refines: ADR-066 (the notio-structured landing), ADR-045/058 (River on lightweight-charts, autoSize + ghost-pane guard). Cross-ref: the Verification law (CLAUDE.md) — verified on a LOCAL PRODUCTION build.

Context: the landing hero's right half read as a **void** — "CAN-MAR 1-0 · LIVE 74'" at the top and "CAN TO WIN 61.4" at the bottom with dead space + a faint glow between, where `features/landing/HeroRiver.tsx`'s Momentum River should be drawing. Not a data problem (the fixture is a live 41→61.4 rise) and not the classic 0-width sizing race (`MomentumRiver` already has `autoSize` + clears its container). Root cause: **HeroRiver passed no `yRange`**, so `MomentumRiver` let lightweight-charts autoscale the price axis to the narrow live band [~41,61.4]; the filled area's gradient (43%→3% opacity) then spread across the whole pane as a faint wash — a river that reads as a void. The working terminal River (`BigRiver`) pins its axis; the hero did not.

Decision: pin `yRange={[0, 100]}` on the hero's `MomentumRiver` (it's a win %, so 0–100 is the natural axis). The area now fills BIG from the value down to 0 instead of a thin autoscaled band — the signature element becomes the hero's visual anchor (design law: "all visual boldness lives in the River"). One-line change; no new component, no motion lib, tokens untouched.

Also: notio (styleui.dev) license is **UNDECLARED** (no `license` in `notio.json`; `/license` + `/terms` 404). The landing structure is a heavily-transformed derivative — added a one-line README attribution beside the existing worldcup26 credit; PROVENANCE already logs it "STRUCTURE STUDY ONLY, zero notio markup ships" (fleet, ADR-066). notio's deps (`next-themes`/`@base-ui`/`usehooks-ts`) are NOT installed (structure adaptation, not a literal `shadcn add`), so nothing to remove.

Consequences / VERIFY (LOCAL PRODUCTION build, main checkout): `pnpm --filter web build` green (18 pages); served on :3000; screenshot `/` and LOOKED — the hero River draws a filled green momentum area rising to the 61.4 edge (the void is gone). Canvas guard: the two VISIBLE hero-River canvases are 460×300 (`cssW 460`) — sized, not the 300px default; two hidden 300×150 canvases (`cssW 0`) are a responsive/ghost pane, not user-visible (same pattern as the terminal's `display:none` mini-charts). Stats count-up targets confirmed as the real facts (104 fixtures / 48 teams / 1,000 credits / Jul 19 MetLife) — the 43/93/898 a human saw were mid-animation frames. axe 0 on `/`. tsc clean.
