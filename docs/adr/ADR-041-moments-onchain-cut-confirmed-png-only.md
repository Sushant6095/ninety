# ADR 041 Moments on-chain mint ‚Äî CUT confirmed after the PNG pipeline; ship PNG-only, saga MOMENTS step = runMomentsStep (apps/worker-jobs/src/moments.ts, Phase 4)

Status: accepted 2026-07-08. Confirms + extends ADR-032. Context: the task was ‚Äî if `MOMENTS_ONCHAIN`, the saga MOMENTS step mints the moment as a cNFT via the chain pkg, stores the sig, and the detail screen shows a Solscan link; TIMEBOX half a day; otherwise `/adr` the cut and ship PNG-only. VERIFY: one minted Moment resolvable via Helius DAS, OR the cut recorded.

Decision ‚Äî the CUT STANDS; ship PNG-only. Within the half-day timebox, the on-chain half's blockers are UNCHANGED from ADR-032 and are all external/provisioning, not code:
- **No `HELIUS_API_KEY`** in this environment ‚Äî the Helius DAS read that the VERIFY hinges on ("minted Moment resolvable via DAS") cannot be exercised.
- **No devnet Bubblegum concurrent-merkle tree** provisioned ‚Äî `mint_moment`'s Bubblegum CPI needs a pre-created tree; creating + funding one is manual devnet setup.
- The Rust CPI already COMPILES (ADR-032, cargo-check) and the swing-card PNG/SVG pipeline is now built (ADR-040), so the ONLY gap is the two external items above + sending the mint tx (a signer/connection) ‚Äî genuinely >half-a-day for a cosmetic feature during the WC26 build. So `MOMENTS_ONCHAIN` stays default OFF.

What shipped (PNG-only, wired + tested): `runMomentsStep(marketId, deps)` in `moments.ts` is the concrete saga MOMENTS-step body ‚Äî it ALWAYS builds the swing-card moment (`buildMoment`, ADR-040: pick swing ‚Üí render SVG ‚Üí upload ‚Üí Moment row), then calls `recordMoment` which, with the flag off, returns `{onchain:false}` (the card IS the moment). The PNG row is persisted BEFORE any mint, so enabling the (currently unwired) mint can never lose the moment. The saga's `mintMoments` hook is documented to wire to `runMomentsStep`. The on-chain seam is ready: flip `MOMENTS_ONCHAIN`, and `recordMoment` will build `mintData(imageUri)` (chain pkg) as the cNFT `metadata_uri` ‚Üí Bubblegum CPI ‚Üí store the returned sig on `Moment.mintSig`; it throws loudly today if flipped prematurely (no tree/key) rather than silently no-op.

Consequences / VERIFY: (+) VERIFY met via the recorded-cut branch (this ADR). PNG-only path green ‚Äî `runMomentsStep` builds + persists the card and reports off-chain with no sig; flat match ‚Üí null; 68 worker-jobs tests, full gate green. (‚àí) To flip on later (all external): provision a devnet Bubblegum tree, set `HELIUS_API_KEY`, wire the mint-tx sender (chain `mintData` + signer/connection) into `recordMoment`, persist `mintSig` on the Moment row, and add the Solscan link to the moment detail screen (`apps/web/src/app/moments/[id]`, currently a stub). Then verify: one minted Moment resolves via Helius DAS. Cross-refs: ADR-032 (original cut), ADR-040 (the swing-card pipeline this step runs), ADR-035 (the saga whose MOMENTS step calls it), ADR-031 (the chain pkg + program that would mint).
