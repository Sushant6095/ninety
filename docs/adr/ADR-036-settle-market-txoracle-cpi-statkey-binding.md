# ADR 036 settle_market on-chain proof gate — CPI txoracle.validate_stat with a two-stat home−away predicate, statKey+FINAL-period binding, fail-closed on unconfirmed encodings (programs/omnipitch_core, prompt 19)

Status: accepted 2026-07-08. Context: prompt 19 — the last real gate of the chain-auth phase. `settle_market` must verify the match result ON-CHAIN before writing it, with NO admin path (CLAUDE.md law). ADR-017 chose Plan A (CPI into TxLINE's txoracle.validate_stat rather than replicating Merkle verification in-program). This ADR records the implementation + the security decisions the proof-auditor forced.

Decision (implemented, cargo check green, proof-auditor re-verified): `settle_market` stays **permissionless + one-shot**, gated ONLY by a CPI to `txoracle.validate_stat`. The new `src/txoracle_cpi.rs` ports the txoracle IDL types (ScoreStat/StatTerm/ProofNode/ScoresBatchSummary/TraderPredicate/Comparison/BinaryExpression — borsh layout matched to `packages/txline/txoracle.json`) and `cpi_validate(...)` builds the instruction (discriminator `[107,197,232,90,191,136,105,185]`), invokes it, and accepts success ONLY when `get_return_data() == Some((TXORACLE_ID, [1u8]))`. The handler enforces, in order: **one-shot** (`require!(m.status < 3, AlreadySettled)` before any write); **C1 market binding** (`fixture_summary.fixture_id == m.match_id` parsed i64, and `epoch_day = m.kickoff / 86_400` derived ON-CHAIN — never from the caller — selecting the roots PDA); **C2 trust-surface pin** (`txoracle_program` pinned via `#[account(address = TXORACLE_ID)]` AND re-checked inside `cpi_validate`; `daily_scores_roots` required `== daily_scores_roots_pda(epoch_day)`, a PDA off TXORACLE_ID so only txoracle can populate it); **C-1 stat identity + C3 extra-time** (four requires: `stat_home.key == STAT_KEY_HOME_GOALS`, `stat_away.key == STAT_KEY_AWAY_GOALS`, and BOTH `period == PERIOD_FINAL`); **C4 predicate soundness** (`comparison_for(result)` maps 1/2/3 → GreaterThan/EqualTo/LessThan on `stat_a − stat_b` vs threshold 0, op = Subtract; result 0 or >3 → InvalidProof).

Key security decision — **stat identity, fail-closed (proof-auditor C-1, CRITICAL).** The first cut only required `stat_home.period == stat_away.period`. That was forgeable: because the predicate is `stat_a − stat_b` with `stat_a = stat_home, stat_b = stat_away`, a caller could pass ANY two genuinely-anchored stats (or the real goal stats SWAPPED) and settle an arbitrary result with 100%-valid proofs — e.g. present away-goals as stat_home and home-goals as stat_away to flip a decisive match, or two equal-valued stats to force a DRAW. Fix: pin WHICH statKey is home and which is away, and require both to be the FINAL period. The predicate now proves specifically `home_goals − away_goals` at full time. This simultaneously closes the **C3 extra-time trap** (H-1): a 90'/half-time snapshot has a non-FINAL period and is rejected, so it cannot settle a match still headed to ET.

Key decision — **DRAW is provable directly.** The txoracle `Comparison` enum has `EqualTo`, so `result = 2` (DRAW) settles via `home − away == 0` (resolves the old proof.rs C4 "equality predicate if available" open question — no synthetic tie-break needed for regulation/ET draws).

Consequences / VERIFY: (+) cargo check green; proof-auditor confirmed no-admin-path, C1/C2/H2/one-shot, and (re-verify) that the statKey+period pin closes the C-1 swap forge and H-1 intermediate-period settle. The whole gate fails in the SAFE direction (see below). (~) **The prompt's headline VERIFY — "localnet/devnet settles a real finished fixture with a Solscan sig" — is BLOCKED here** and NOT faked: it needs (a) the txoracle program present on the target cluster (clone/mock on localnet, or devnet), and (b) a real proof bundle for a FINISHED fixture whose values are anchored in `daily_scores_roots`. Neither is available in this environment. The CPI wiring, account pinning, binding, and predicate are structurally verified by review + compile; the live settle lands when the cluster has txoracle + a real root (couples with the ADR-035 saga `sendSettle` impl).

**Fail-closed via IMPOSSIBLE SENTINELS (settlement is DISABLED until STEP-0).** The first fix pinned `1002/1003/period-4` and claimed "unconfirmed ⇒ fails closed, never forges." The proof-auditor REFUTED that on re-verify, and correctly: ADR-017/ADR-015 observed `1002/1003` are REAL anchored stats that are NOT goals (live: `1002=2, 1003=1` while the score was `1–3`). Pinning a real-but-wrong key does NOT fail closed — an attacker submits a GENUINE Merkle proof for those real leaves, all four requires pass, and the `home−away` sign comes out wrong → forged result. A plausible-but-real wrong constant is strictly MORE dangerous than an impossible one. So the constants are now IMPOSSIBLE SENTINELS: `STAT_KEY_HOME_GOALS = u32::MAX`, `STAT_KEY_AWAY_GOALS = u32::MAX-1`, `PERIOD_FINAL = i32::MAX`. No leaf in `daily_scores_roots` is keyed that way, so txoracle can never validate a proof for them — EVERY settle is rejected. This is the genuine fail-closed state: settlement is wired + reviewed but cannot settle anything (forged or legit) until STEP-0 replaces the three sentinels with the real encodings. Going live = editing three constants after STEP-0 confirms them.

OPEN / STEP-0 (must resolve — and replace the sentinels — before real settlement):
1. **Confirm the FINAL home/away-goals statKeys and the FINISHED period value** against a real anchored fixture, then replace the `u32::MAX / i32::MAX` sentinels. (NOT 1002/1003 — those are confirmed non-goals.)
2. **Terminal-period uniqueness.** The FINAL period must be *terminal and uniquely valued*: if the daily tree anchors more than one `(home-goals, FINAL)` leaf with differing values (e.g. a stoppage-time goal snapshot also marked "final period"), an attacker could prove the earlier/lower value. Confirm exactly one FINAL home/away-goals value is anchored per fixture.
3. **Zero-goal-side liveness.** Both requires demand a genuine `FINAL` leaf per team. If the feed does not anchor a `0`-goals leaf for a side (a 0–0, or the losing side of a 2–0), that side has no provable stat → the match is unsettleable. Confirm both teams' final-goals leaves — including zeros — are always anchored.
4. **Penalty-shootout result mapping.** A shootout win has level goals after ET, so a goals-only `home − away` predicate returns DRAW (and the period pin makes the wrong-draw path concrete). Needs the shootout/decision stat consumed as an extra term, or a distinct settle path for shootout ties. (Not needed for group-stage WC26; needed for knockouts.)
5. **epochDay boundary** — `m.kickoff / 86_400` (UTC day of kickoff). Confirm txoracle anchors a finished fixture's FINAL stats under the kickoff-day root even when full time crosses midnight UTC; if it uses the finish day, derive epoch_day from a finish timestamp instead.

Follow-ups (separate, non-blocking): **M-1** — `initialize_market` is permissionless with a caller-supplied immutable `kickoff`; an attacker can front-run and squat a real `match_id`'s PDA with a wrong `kickoff` (or non-numeric `match_id`), making that market permanently unsettleable (liveness DoS, not mis-settle). Gate market creation or validate kickoff. **L-2** — the old `proof::verify_txline_proof` stub is now dead (settle gates via `txoracle_cpi::cpi_validate`); delete it and update `programs/CLAUDE.md` wording so the gate name matches reality. **L-3** — `packages/chain/src/builders.ts::settleData` still encodes the old `(result, proof)` arg shape and is now STALE (marked in-file); rebuild it as the structured-args borsh encoder as part of the STEP-0 saga `sendSettle` live-wiring — dead until then since settlement is fail-closed. (Helius detection unaffected — keys on discriminator + market PDA.)
