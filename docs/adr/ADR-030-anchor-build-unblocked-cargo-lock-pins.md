# ADR 030 anchor `build-sbf` UNBLOCKED ‚Äî six Cargo.lock pins drop the edition2024/MSRV crates out of the SBF graph ‚Üí `anchor build` clean, `.so` + IDL produced (resolves ADR-017/ADR-029)

Status: accepted 2026-07-08. Context: ADR-017 (and ADR-029 same-day) recorded the on-chain build as blocked ‚Äî `anchor build` ‚Üí `cargo build-sbf` uses platform-tools v1.43 / rustc 1.79, which cannot parse `edition2024` transitive deps. ADR-029 concluded the interim VERIFY had to be `cargo check` (no deployable `.so`). That is now SUPERSEDED: the blocker is fixed without changing platform-tools, using the "pinned Cargo.lock" workaround ADR-017 itself named. Decision: pin every recently-published transitive crate that adopted edition2024 (or raised its MSRV above 1.79) DOWN to its last 1.79-compatible version, in `programs/omnipitch_core/Cargo.lock` (committed). The six offenders, each found by iterating `anchor build` and pinning the crate the error named: (1) `block-buffer 0.12.1 ‚Üí 0.10.4` ‚Äî pulled via `blake3 1.8.5 ‚Üí digest 0.11.3`; actually pinned by dropping `blake3 1.8.5 ‚Üí 1.5.5` (solana-program's `blake3 = "^1.5.0"` allows it; 1.5.x uses digest 0.10 / block-buffer 0.10). (2) `zeroize_derive 1.5.0 ‚Üí 1.4.2`. (3) the toml stack via `proc-macro-crate 3.5.0 ‚Üí 3.1.0` (drags `toml_edit 0.25‚Üí0.21`, `toml_datetime 1.1‚Üí0.6`, `winnow 1.0‚Üí0.5`) ‚Äî pulled through `borsh-derive 1.7`. (4) `indexmap 2.14.0 ‚Üí 2.6.0` (drags `hashbrown 0.17‚Üí0.15`). (5) `jobserver 0.1.35 ‚Üí 0.1.32` (MSRV 1.85). (6) `unicode-segmentation 1.13.3 ‚Üí 1.12.0` (MSRV 1.85). Plus the standard Anchor 0.30 IDL config: add `[features] idl-build = ["anchor-lang/idl-build"]` to the program crate's Cargo.toml. Two earlier config bugs were fixed first (ADR-029): the placeholder `declare_id!`/Anchor.toml program-id (‚Üí real id `6ps8ao7CVhacnRajvFXWTmkknsRnHfEbWmtQ3nDCdBkj`) and the missing `overflow-checks`. Result: `anchor build` exits 0 and produces `target/deploy/omnipitch_core.so` (203 KB) + `target/idl/omnipitch_core.json`. Consequences: (+) THE ON-CHAIN PHASE IS UNBLOCKED ‚Äî `anchor build`/`anchor deploy`/`anchor test` all run; the on-chain prompts (settle, claim_points, moments, negative-path) can now reach their real VERIFY (localnet/devnet), not just `cargo check`. Devnet wallet is funded (1.99 SOL), so devnet deploy is available too. (+) No platform-tools change needed ‚Üí reproducible on this machine; the committed Cargo.lock IS the pin (future `anchor build` respects it ‚Äî do NOT `cargo update` these six crates or the blocker returns; use `--locked` in CI). (~) The pins are transitive-only (no direct dependency downgrade, no API change to our program or to anchor-lang 0.30.1); they only force older builds of build-time/crypto utility crates. (‚àí) A future `anchor-lang`/`solana-program` bump may reintroduce newer transitive deps ‚Üí re-pin or (the cleaner long-term fix) install a newer platform-tools with rustc ‚â• 1.85 and drop the pins.
