# ADR 026 order command вҖ” risk gate вҶ’ LMSR fill at live b вҶ’ burned fee вҶ’ position + credit-ledger effects (apps/api/src/engine/order.ts, index.ts)

Status: accepted 2026-07-08. Context: ADR-025 shipped the engine runtime but left the `order` `EngineCmd` kind as an explicit follow-up ("an order against a halted market MUST reject `MARKET_HALTED`, not queue"). This ADR adds it. Decision (implemented, gate-green, engine-guardian 6/6): (1) PURE ORDER CORE (`order.ts`) вҖ” `applyOrder(input)вҶ’OrderResult` is a pure, deterministic function: given `(side, outcome, size, live q, live b, tradeable, balance, position, recentOrderTimes, now, limit?)` it returns a typed fill or a typed reject. No IO, no state ownership, no wall clock (`now` is event-time). RISK GATE (fail-fast, in order): `INVALID_SIZE` (size not a positive finite integer вҖ” 0/negative/NaN/ВұInfinity/fractional); `INVALID_OUTCOME` (outcome not an in-range integer index into q); `MARKET_HALTED` (`!canAcceptOrder` вҖ” only OPEN/LIVE trade); `RATE_LIMIT` (>5 accepted orders in the trailing 1s for this user, counted from `recentOrderTimes` vs `now`); then per-side price checks. FILL uses `amm.buyCost(q,b,outcome,Вұsize)` at the LIVE server b вҖ” LMSR math stays in `amm.ts` (purity law); buy debits `cost+fee`, sell refunds `costвҲ’fee`. FEE = 1% of the moved cost, BANKER'S-ROUNDED to whole credits and BURNED (removed from supply) вҖ” banker's (round-half-to-even) chosen over floor/ceil so rounding drifts toward neither user nor house across many trades. Ledger events: buy вҶ’ `[debit(cost+fee), burn(fee)]`, sell вҶ’ `[credit(refundвҲ’fee), burn(fee)]`. `positionDelta`/`shareDelta` = Вұsize. (2) DECISIONS the task required be explicit: EPSILON POLICY вҖ” `BALANCE_EPS = 1e-9`; a buy is rejected `INSUFFICIENT_BALANCE` only if `debit вҲ’ balance > 1e-9` (a shortfall of exactly 1e-9 is treated as float error and FILLS; 1e-8 rejects) вҖ” LMSR cost is a float, so an exact-equality balance check would spuriously reject. SERVER PRICE WINS вҖ” `applyOrder` always prices at the passed-in live q/b, never a client quote; the optional `limit` is a client slippage guard (buy: max total debit; sell: min net credit) вҶ’ `SLIPPAGE` reject when the server price has moved past it, and the client reconciles by re-quoting/resubmitting. POSITION CAP = 5000 per outcome, boundary inclusive (вҶ’5000 fills, вҶ’5001 rejects); no shorting (sell > held вҶ’ `INSUFFICIENT_POSITION`). (3) ENGINE INTEGRATION (`index.ts`) вҖ” `EngineMarket` gains `q: number[]` (seeded `[0,0,0]`, 1X2 v1) and `positions: Record<string, number[]>`, both part of the JOURNALED state so replay/snapshot reproduce them exactly. New `{kind:"order"; at; order: OrderCmd}` `EngineCmd`; `applyOrderCmd` delegates to `applyOrder` and applies the q + position deltas immutably (spread-copy, no mutation), emitting `fill`/`position`/`ledger` effects вҖ” or a single `reject` effect with NO state change. `EngineDeps.emit` widened to `EngineEffect = MarketEffect | fill | position | ledger | reject`. The fill b is `bHint / max(spread Г— spreadAt(lifecycle, at), 0.01)` вҖ” compounding BOTH spread sources: the cortex hazard spread (from `prices.marks`) AND the ADR-005 reopen decay (3Г—вҶ’1Г— over 60s after a goal-halt reopen), floored so a malformed mark can't div-by-0. Balance authority stays the off-chain ledger (ADR-003): `balance` + `recentOrderTimes` are supplied BY THE CALLER inside the journaled `OrderCmd` (so replay is deterministic), NOT owned by the per-market engine. Consequences / VERIFY: (+) engine-guardian 6/6 invariants PASS (single-writer, journal-then-ack, deterministic time-from-events, amm purity, no-http-ws boundary, MARKET_HALTED). Findings FIXED with regression tests before shipping вҖ” CRITICAL C1: `outcome` was unvalidated вҶ’ an out-of-range index made `buyCost` return NaN (bricking the market forever, durably + on replay) or 0 (free fill / phantom position); added `INVALID_OUTCOME` gate before any amm math. HIGH H1: the fill b now applies the ADR-005 reopen spread decay via `spreadAt()` (previously only the mark spread, leaving the post-goal adverse-selection window fillable at full-tight liquidity); the comment no longer overclaims. LOW: `b` divisor floored against a bad mark (div-by-0); redundant `Math.abs` on the buy cap dropped (position вүҘ 0, no shorting) and the unreachable sell-side cap check removed (`INSUFFICIENT_POSITION` already guarantees `0 вүӨ positionвҲ’size вүӨ CAP`). (+) VERIFY green вҖ” order cost matches an INDEPENDENT closed-form LMSR to < 1e-9 across 3 fixed + 500 seeded-random cases; sell refund == buy cost (path-independence); a buy+sell round-trip reduces total credits by EXACTLY the two burned fees; banker's rounding table; every reject unit-tested (INVALID_SIZE, INVALID_OUTCOME, MARKET_HALTED, RATE_LIMIT boundary [5th fills / 6th rejects / stale fall out of the window], INSUFFICIENT_BALANCE at the 1e-9 epsilon boundary, POSITION_CAP boundary, INSUFFICIENT_POSITION, SLIPPAGE on a moved server b); engine-loop tests (order fills вҶ’ q + position advance + fill/position/ledger effects; halted market вҶ’ single `reject`, no state change; deterministic replay reproduces q/positions from the command stream AND from a `snapshotEvery:1` snapshot; reopen-decay makes a post-goal fill dearer than at plain LIVE). Full gate 9/9 (build/test/lint), 102 api tests. (~) No CLAUDE.md law or hook changed; ENFORCES the existing laws (amm-purity вҖ” no LMSR in the loop; MARKET_HALTED; determinism; journal-then-ack via the ADR-025 pipeline). Play-money invariant unaffected вҖ” the fee is BURNED, never a payout; NO deposits/cashout; NO admin/result path. (вҲ’) Deferred follow-ups (engine-guardian MEDIUM/LOW, v1-acceptable for a play-money core; recorded so they are not re-litigated): LEDGER IS THE ATOMIC BALANCE AUTHORITY (H2) вҖ” because `balance` is a caller snapshot and the per-market journal serializes only one market, the engine's `INSUFFICIENT_BALANCE` gate is ADVISORY: two concurrent orders on different markets (or rapid-fire on one lane) can each pass against the same snapshot and over-debit, and the engine commits q + position BEFORE the authoritative debit. The ledger MUST be the atomic authority for the debit and MUST own a compensation path (reverse the fill / unwind q + position) when it rejects a debit the engine already applied; the gateway owns the cross-market rate window. This boundary is deliberate (owning cross-market balance in the engine would break single-writer-per-market). EFFECT OUTBOX (H3, inherited ADR-025) вҖ” a crash between journal-append and emit recovers q + position via replay but does NOT re-emit the `fill`/`debit`/`burn`, so the engine can hold shares the ledger never saw debited; the ADR-025 at-least-once outbox upgrade now carries money and must land before this trades live. LEDGER EVENT CONTRACT (M1) вҖ” `burn` is SUPPLY accounting, NOT a second balance debit; a consumer that debits on both `debit` and `burn` charges `cost + 2В·fee`. Lock this in the cross-service schema before wiring a ledger consumer. POSITIONS PRUNING (L3) вҖ” `positions` grows one entry per user per market and is fully snapshotted; bound/prune for popular markets. N-OUTCOME q вҖ” q is fixed length 3 (1X2) in v1; sizing an n-outcome q from the mark's fair vector is future. The emitвҶ’bus/ledger topic mapping and the order SOURCE (a gateway http/ws surface feeding `OrderCmd`s with authoritative balance + rate window) are out of this engine-scope session. Cross-refs: ADR-025 (the runtime this order command plugs into; its journal replays q/positions and the outbox follow-up now carries money), ADR-024 (the lifecycle `canAcceptOrder` gate), ADR-005 (halt/reopen decaying spread now applied to the fill b via `spreadAt`), ADR-002 (LMSR cost the fill uses), ADR-003 (off-chain credit ledger = the balance authority these events feed).
