# ADR 013 txline client transport-injectable auth flow

Status: accepted 2026-07-07. Context: `packages/txline` is the ONLY code allowed to call TxLINE, and only `packages/chain` may build Solana txs (CLAUDE.md law). The client had to implement the TXLINE-MAP §0 auth flow — guest JWT → on-chain `txoracle.subscribe(level, weeks)` → sign `"{txSig}:{leagues}:{jwt}"` → `/api/token/activate` → apiToken, then both headers on every data call — but cannot reach the live devnet API in dev/CI (no funded wallet, no creds, no network), and F1's fixtures path is still ⚠ unconfirmed. Decision: (1) `TxLineClient` (`packages/txline/src/client.ts`) is transport-injectable — it takes a `fetch` (default global fetch), a `Subscriber` (builds+sends the on-chain subscribe tx, returns the signature) and a `MessageSigner` (signs the activation message); it never imports `@solana/web3.js`, so the subscribe tx is built in `packages/chain` (`buildSubscribeIx`, anchor discriminator + level/weeks; account list & arg encoding ⚠ pending the txoracle IDL), honoring the law. (2) Network safety: a NETWORKS registry (`packages/txline/src/network.ts`) pins each cluster to exactly one apiOrigin (devnet→txline-dev.txodds.com, mainnet-beta→txline.txodds.com); `assertOriginMatchesCluster` runs in the constructor and throws on any mismatch, so a devnet subscribe tx can never activate against the mainnet origin (TXLINE-MAP §0). (3) Session: one cached `{jwt, apiToken}`; concurrent handshakes are coalesced (the subscribe step is an on-chain tx — never run it twice); a proactive TTL re-auth window; and a single refresh-and-retry on 401/403 — every data request carries BOTH headers (`Authorization: Bearer {jwt}` + `X-Api-Token: {apiToken}`). (4) Verification without infra: a deterministic in-process mock (`packages/txline/src/mock.ts`, real WC26 sample data) drives the full handshake; the verify script (`packages/txline/scripts/snapshot.ts`) prints an authenticated fixtures/scores snapshot for a real WC26 fixture and self-asserts (dual headers, cache, refresh, network guard), doubles as `pnpm --filter @omnipitch/txline test`, and `--emit` writes `docs/txline-samples/*.json`. (5) Typed endpoint wrappers so far: `scoresSnapshot` (S1, confirmed path) and `fixtures` (F1, path `/api/scores/schedule` ⚠ pending Day-0 confirmation from examples/fetching-snapshots). Consequences: (+) the client is testable and law-compliant with no live dependency — production uses global fetch against the live API when creds exist; (+) cross-network activation is structurally impossible; (+) the on-chain subscribe auth cost happens once and is coalesced/cached. (−) two auth-flow specifics stay placeholders until Day-0 — the F1 fixtures path, and the txoracle subscribe instruction's exact accounts/arg types (both marked ⚠ in code and TXLINE-MAP). No change to any CLAUDE.md law or hook. Cross-ref: ADR-006 (auth), TXLINE-MAP Day-0 checklist.
