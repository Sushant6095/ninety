# ADR 048 apps/web ships to Vercel as a standalone fixture-data prototype; live-backend/Solana wiring stays for the global-track branch

Status: accepted 2026-07-13. Context: the hackathon submission is judged from a live public URL, so apps/web needed a production deploy while main stays deployable. apps/web is fixture-wired today (ADR-047: `lib/api.ts` has 0 importers, `lib/fixtures.ts` powers 23 files) and imports zero `@omnipitch/*` packages, so a real-backend deploy would drag in the monorepo (apps/api's native uWS binary, workspace resolution) for no runtime gain. This is a deploy/positioning decision — no service, engine, or bus behaviour moves; SETTLEMENT_LIVE stays false, no admin result path, no chain writes in web (architecture law upheld).

Decision:
- **Standalone deploy.** apps/web builds and deploys ALONE (Vercel project `sushant6095s-projects/ninety`, Root Directory `apps/web`) at LIVE URL https://ninety-nu.vercel.app — production, public, all 15 routes 200. The unused `@omnipitch/schema` workspace dep was removed so there is no workspace resolution and no native-binary build; re-add it on the global-track branch when the live client actually imports it.
- **npm install, self-contained tsconfig.** A standalone upload carries no `packageManager` pin/lockfile, so Vercel's baseline pnpm hit the Node-undici registry bug (`ERR_INVALID_THIS`); `vercel.json` installCommand is `npm install --legacy-peer-deps` (no `workspace:` deps remain, so npm resolves). Pinned pnpm@9.15.9 + Node 20.x in package.json; inlined `packages/config/tsconfig.base.json` into apps/web tsconfig (absent in a standalone upload → TS5083, kept `strict:true`); added `@types/node` + `@types/react-dom` as direct devDeps.
- **Library law + copy law fixed pre-deploy.** Removed the banned second chart lib `recharts` (PortfolioCard reuses the EquityCurve inline-SVG spark; AttackMomentum is a hand-rolled signed-bar SVG, per the ADR-045 row-viz precedent — lightweight-charts stays reserved for the hero River). Replaced "odds" in how-it-works product copy with "prices"/"a live probability" (literal `/api/odds/*` endpoint paths untouched, per ADR-047's reasoning).
- **Honest prototype ribbon.** A slim, token-only, non-dismissible ribbon on every route states the build is fixture data with live API & Solana wiring queued for the global track, and repeats the play-money disclaimer — it does not fake liveness.

Consequences: the submission URL is live and does not overstate what runs. The 7-step live-wiring plan (real `/markets`, WS, quote, read-only Solana ProofBadge) is deliberately NOT started and stays for a separate global-track branch off ADR-042's endpoints. FOLLOW-UP (parameter tweak, within envelope): bump apps/web engines to Node 24.x before 2026-10-01 (Vercel deprecates Node 20 for deploys on/after that date; build is green on Node 20 today and the submission window is well before it). VERIFY: production URL returns 200 on all 15 routes; `grep recharts apps/web/package.json` is empty; no bet/stake/odds/wager in apps/web product copy outside literal endpoint paths; design-cop passes home.png + terminal.png at sm/md/lg/xl.
