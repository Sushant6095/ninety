# ADR 025 engine runtime — durable journal (journal-then-ack) + per-market serialized loop (apps/api/src/engine/journal.ts, index.ts)

Status: accepted 2026-07-07. Context: ADR-024 defined the market lifecycle as a pure `transition(state,cmd)→{state,effects}` reducer but left its runtime PENDING — durability and the single-writer bus wiring (match.events → apply → publish; prices.marks → AMM re-anchor). This ADR fuses the two runtime tasks shipped this session (both gate-green, engine-guardian-reviewed 6/6). Decision (implemented): (1) DURABLE JOURNAL (`journal.ts`) — journal-then-ack: a command is appended durably BEFORE it is applied, so a crash between append and apply is recovered by replay and applied EXACTLY ONCE. Each entry carries a per-key monotone seq `n`; a snapshot is taken every N events (default 10k); `recover()` = latest snapshot + replay of the contiguous tail. Storage is injected behind a `JournalStore` port so the reducer logic stays pure and property-testable — `MemJournalStore` (tests) + `RedisJournalStore` (prod; the Redis client is INJECTED and the file uses only `import type { Redis }`, so no runtime redis import leaks). Per ADR-020 the engine journal is the named raw-Redis exception to the bus-only law. HARD ERRORS, never silent: `n`-contiguity gap detection — a journal trimmed below the snapshot or a lost head throws `JournalError("gap")`; a torn/corrupt snapshot (stored hash ≠ recomputed) throws `JournalError("corrupt-snapshot")`; snapshots are one atomic write (no torn state). `hashState()` is a deterministic content hash (sorted keys, undefined-key elision so an in-memory state and its JSON-round-trip hash identically) for recover-vs-live equivalence. (2) PER-MARKET SERIALIZED LOOP (`index.ts`) — one logical writer per market via a promise-chain lane → strict arrival order across BOTH `prices.marks` (AMM re-anchor) and `match.events` (lifecycle). Pipeline per command: validate → JOURNAL → apply → emit. Commands arriving during a market's recovery queue behind the recover head (never dropped); a queue-depth cap yields a typed BACKPRESSURE rejection + metric. Single-writer PROCESS is enforced by a Redis `SET NX PX` lease (`RedisLease` + renew loop) — a second engine exits loudly (`EngineLeaseError`). AMM tether: marks re-anchor `bHint`/spread; `amm.ts` stays pure (no LMSR in the loop). Metrics: processed / backpressureRejects / maxDepth / unserveable. Consequences / VERIFY: (+) engine-guardian 6/6 invariants PASS (single-writer, journal-then-ack, deterministic time-from-events, amm purity, boundary/no-http-ws + injected type-only Redis, halted-rejects-orders); 3 findings FIXED with regression tests before shipping — CRITICAL: recovery now FAILS SAFE — a recover failure of ANY kind (a `JournalError` OR a transient store error) quarantines the market (refuse to serve, alert, `metrics.unserveable`); previously a non-`JournalError` fell through to a fresh-seeded WRONG state and polluted the journal with a duplicate `n=0`. HIGH ×2 (end-to-end journal-then-ack in the bus adapter): `submit()` now returns a `durable` promise the bus consume handler awaits before acking (the source is never acked before the command is durable), and it nacks/redelivers on BACKPRESSURE (previously the rejection was discarded and the message acked-and-lost). LOW: the replay-seed and live-seed are now identical by construction (both seed with `marketId` as `matchId`); a quarantined lane returns a typed UNSERVEABLE, not a false accept. (+) VERIFY green — journal property tests (10.5k commands: replay reproduces the identical state hash; crash-simulated recovery == live; crash-after-append-before-apply applies exactly once; torn-snapshot / trimmed-below-snapshot / empty-journal edges are hard errors) + engine-loop tests (2 markets × 1k interleaved commands → per-market ordering perfect; marks/lifecycle interleave in strict arrival order; BACKPRESSURE cap; second-process lease exit; commands-queued-during-recovery; quarantine on recovery failure incl. a non-`JournalError`). Full gate 9/9; engine-guardian 6/6. (~) No CLAUDE.md law or hook changed: the runtime ENFORCES the existing single-writer / journal-then-ack / no-http-ws / amm-purity / MARKET_HALTED laws; the engine journal is the raw-Redis exception ALREADY named in ADR-020's amended Architecture law. Play-money invariant unaffected; NO admin/result path added. (−) Deferred follow-ups (engine-guardian rated MEDIUM/LOW, v1-acceptable for a play-money core — upgrade paths recorded so they are not re-litigated): EFFECT OUTBOX (MEDIUM) — a crashed/failed emit is logged but not re-emitted and recovery does not re-emit, so a `settled`/`halt`/`void` emission can be lost; upgrade = journal the effect and pump it at-least-once with idempotent consumers. LEASE FENCING (MEDIUM) — renew is a plain `SET XX` with no fencing token; a process paused past the TTL could split-brain; upgrade = owner-checked Lua renew + a fencing token on writes. TAIL-TRUNCATION WATERMARK (LOW) — gap detection catches holes/lost-head but not a pure loss of the NEWEST entries; upgrade = persist a monotone max-`n` high-water mark and assert the tail reaches it. CROSS-MARKET CONSUMER COUPLING — one bus consumer awaiting per-command durability serializes the topic across markets; upgrade = partition consumers per market. The emit topic-mapping is a stub and the `order` `EngineCmd` kind is future — an order against a halted market MUST reject `MARKET_HALTED`, not queue. Cross-refs: ADR-024 (the market reducer this journals/replays), ADR-005 (halt/reopen/decaying spread the loop drives), ADR-020 (bus law — the engine journal is its named exception), ADR-011 (RedisBus delivery semantics under the bus adapter).
