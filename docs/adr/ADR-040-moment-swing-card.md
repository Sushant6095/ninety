# ADR 040 Moment swing card ‚Äî biggest-|Œîmark| window ‚Üí server-rendered SVG card (river segment, token palette, mono) ‚Üí upload ‚Üí Moment row (apps/worker-jobs/src/swing-card.ts, Phase 4)

Status: accepted 2026-07-08. Context: each settled match should yield ONE shareable "Moment" card of its biggest market swing (the Momentum River is the product's signature element). Task called for a PNG via satori/canvas, uploaded to R2, with a Moment row the gallery consumes.

Decision (implemented; 66 worker-jobs tests green):
- **`pickSwingWindow(samples)`** ‚Äî pure, O(n¬∑outcomes): the biggest TIME-ORDERED directional move (max rise or fall) across all outcomes over the whole match, tracking the running min/max index per outcome. Returns `{outcome, from, to, delta (signed), minuteFrom, minuteTo}` in one-decimal price points (fair√ó100).
- **`renderSwingCardSvg(samples, win, meta)`** ‚Äî a self-contained **SVG** string: the outcome's price series as a Momentum-River polyline (colored `--up`/`--down` by direction), the swing delta + from‚Üíto + minute range in **IBM Plex Mono tabular** numbers, on the token palette (mirrors `design/tokens.css`). All values escaped.
- **`buildMoment(marketId, deps)`** ‚Äî orchestrates pick ‚Üí render ‚Üí upload ‚Üí persist; returns null on a flat match. All IO is injected via `MomentDeps { getMatchMarks, uploadImage, saveMoment, getMeta? }`, matching the saga's DI pattern (worker-jobs has no DB/R2 client wired yet ‚Äî that's the same follow-up as ADR-035's Prisma saga deps).

**Decision ‚Äî SVG, not satori/canvas (deviation from the literal task):** the card is a data-viz (a price polyline + numbers), for which a hand-written SVG is the correct, dependency-free primitive ‚Äî satori (HTML/CSS‚ÜíSVG) adds nothing here, and `canvas`/`@resvg/resvg-js` are native deps that don't build reliably in this environment. SVG is server-rendered, renders natively in browsers/OG-image contexts, and is what the uploader stores (`moments/{marketId}.svg`). Literal PNG rasterization (resvg SVG‚ÜíPNG) is a one-line last mile behind the uploader, added when the dep + R2 creds are provisioned; nothing downstream depends on the raster format (the gallery `<img>` handles SVG).

Consequences / VERIFY: (+) a replayed match yields exactly ONE card whose minute + delta match the data ‚Äî `pickSwingWindow` unit tests (biggest rise/fall, signed delta, null on flat) + `buildMoment` (one row, `win` matches, uploaded + persisted exactly once, null‚Üíno upload/row); `renderSwingCardSvg` cites the delta (`+13.0`), `25' ‚Üí 40'`, `17.0 ‚Üí 30.0`, `--up`/`--down` by sign, IBM Plex Mono, the river polyline.
