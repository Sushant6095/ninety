# ADR 019 ingest stream gap recovery + exactly-once goal-derivation hardening

Extends ADR-018 (2026-07-07). Status: accepted 2026-07-07. Context: ADR-018 stood up worker-ingest normalizing the txline SSE scores/odds streams into canonical Envelopes with `(source, source_seq)` idempotency, but its first VERIFY-green suite exercised only the deterministic happy path — a clean, gapless, in-order stream. Two follow-on pressures forced this ADR: (a) the live devnet scores stream is sparse and emits mostly `{Ts}` keepalives (ADR-016), so a silent-but-alive stream is indistinguishable from a real stall without extra signal, and a dropped SSE connection silently loses any goals that occurred during the gap; and (b) an adversarial multi-agent review of the goal-derivation path found 6 confirmed defects the green happy-path tests never touched. Both fixes are implemented and gate-green in `apps/worker-ingest` + `packages/txline`. Decision — gap recovery: (1) per-stream heartbeat watchdog — `packages/txline` `StreamOptions` gained an `onHeartbeat?: () => void` callback the SSE client fires on every skipped keepalive/comment frame (only txline sees raw SSE frames — monorepo law — and it silently skips keepalives, so surfacing them lets the ingest watchdog tell an alive-but-quiet stream from a stall); a gap fires on either `>silenceMs` (default 20s) with no frame at all, or the stream ending/erroring (connection drop). (2) on a gap during LIVE, recovery reconciles per ACTIVE fixture (fetch scores/odds snapshot for each fixtureId seen so far) and re-emits the CANONICAL events flagged `recovered: true` in payload — NOT synthetic events under a new source like `txline.snapshot`; they keep the canonical `source`/`source_seq` so they dedupe against any live counterpart, and the `recovered` flag only survives on truly-missed events. (3) `feed.gap` publishes as a system/ops event to a dedicated Redis stream `feed.gap.v1`, deliberately OUTSIDE the typed `AnyEvent` bus — the Envelope `type` enum has no system/fixture variant, so a gap alert is not a domain event. GOVERNANCE POINT: this establishes a system-event channel distinct from the domain bus; proposed CLAUDE.md edit below. (4) reconnect uses exponential backoff + jitter, reset after any stream that made progress; the watchdog uses a cleared-on-progress real `setTimeout` (not `clock.sleep`) so no dangling timers, and `select`s [pending `next()`, timeout, abort] so `stop()` is prompt; the clock is injectable for deterministic tests. Decision — exactly-once goal hardening (the 6 review findings): (5) score goal `source_seq` is now GLOBALLY UNIQUE per goal: `baseSeq*8 + (away?4:0) + goalIndex`; the old `baseSeq`/`baseSeq+1` scheme let an away goal at stream-Seq N collide with the next state's home goal at Seq N+1 (Seq is +1-monotonic), dropping a real goal as a false duplicate — the block-per-Seq scheme keeps the schema's `(source, source_seq)` idempotency contract intact with no `match_id` in the key. (6) `normalizeScore` emits ONE event per goal in the delta (loop over `hg-phg` / `ag-pag`), so a multi-goal recovery snapshot no longer collapses several goals into one event; `payload.score` stays the final cumulative scoreline (intermediate scorelines are unknowable from a snapshot diff). (7) monotonicity guard in `ingestScore`: a score state whose Seq `<=` the last-seen Seq for that fixture is dropped (no diff, no `prevScore` update) — without it a lagging/replayed recovery snapshot regressed `prevScore` and the next live frame re-derived an already-published goal with a fresh seq → duplicate. (8) journal-then-ack: the dedup mark (`dedup.add`) and `prevScore.set` now happen only AFTER a successful `bus.publish`; previously they mutated first, so a publish failure advanced state while the goal reached no consumer AND recovery couldn't re-derive it → silent permanent goal loss — this aligns ingest with the engine's journal-then-ack law. (9) supervisor resource fixes: the `catch` (onEvent-throw) path now closes the stream generator like every other exit (no leaked SSE socket under a bus outage); one abort listener per supervisor instead of one per reconnect (no MaxListeners leak on a flapping feed); backoff races the abort signal so `stop()` doesn't hang out a full backoff window. Consequences: (+) worker-ingest survives connection drops and quiet-but-alive streams without losing goals, and re-derivation is duplicate-safe under replay/lag; (+) VERIFY green in `apps/worker-ingest` + `packages/txline`. Worth stating: the first green suite passed on the deterministic happy path while all 6 of these defects were live — happy-path green is not correctness; the adversarial review is what surfaced them. (~) goal de-dup now rests primarily on the `prevScore` diff + monotonicity guard, with `(source, source_seq)` as a secondary net. (−) known accepted edges, documented with `ponytail:` comments in code: a rare VAR disallow-then-rescore to the exact same scoreline, and a partial publish failure mid multi-goal-recovery delta. Play-money invariant unaffected; NO admin/result path added. CLAUDE.md governance (finding 3): `feed.gap.v1` sits at the edge of the Architecture law "Events only: services communicate via `packages/bus` using types from `packages/schema`" — it is a Redis stream that is intentionally NOT a `packages/schema` `AnyEvent`. Proposed exact edit — add one bullet under `## Architecture law`: "- System/ops signals (e.g. `feed.gap.v1`) may use a dedicated Redis stream OUTSIDE the typed `AnyEvent` bus; they are NOT domain events and MUST NOT be added to the `packages/schema` Envelope union. Domain events still flow only through `packages/bus` with `packages/schema` types." Proposal only — no hook or CLAUDE.md law is edited by this ADR (CLAUDE.md changes need human sign-off). Cross-refs ADR-018 (ingest normalization, extended here), ADR-016 (keepalive-sparse scores stream), ADR-013 (txline client), ADR-011 (bus delivery semantics), TXLINE-MAP §2.
