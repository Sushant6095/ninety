# ADR 058 Terminal-craft pass — self-healing autoSize River, screener rail, one press vocabulary, thrash gap filed

Status: accepted 2026-07-15 (`merge/live-integration`). Cross-ref: ADR-045 (River on lightweight-charts — this hardens its sizing), ADR-052 (animation libs — the press/tooltip vocabulary lives inside them), ADR-057 (design-law pass + MotionScore posture — extended by this session), ADR-049 (feeling criterion).

Context: the hero Momentum River rendered as a black void on the live deploy (canvases at the 300×150 HTML default / 0×0) — root cause was the pre-autoSize one-shot `width: el.clientWidth` race; one production load also showed duplicate unsized ghost panes beside live ones. Separately: hyperscreener.asxn.xyz was studied for rail structure, emil-design-eng plans 005/006 ran on interaction craft, and MotionScore was re-audited fresh against the live deploy.

Decision:
- **The River sizes itself; containers can never host two panes.** `createChart` uses `autoSize` (commit d499b7a; context7-verified lightweight-charts v4 docs: autoSize hands sizing to the library's own ResizeObserver, `resize()` is ignored under it) — the one-shot clientWidth race is gone. MomentumRiver clears its container before `createChart` (commit 2c9cb3d), so whatever races us, the component self-heals; ghost panes are impossible by construction. Verified 5/5 fresh throttled loads on PRODUCTION plus a visual check — River sizing claims verify on the live deploy, never localhost (a prior session falsely reported this verified).
- **The terminal's left rail is a market screener; sorting is deliberately omitted.** hyperscreener studied for STRUCTURE only (dense table, filter pills, in-card search) — zero markup copied, rebuilt in Ninety tokens (commit f6017d6). CompetitionsRail: filter pills with live counts (All/Live/Faves), a real column header row (UTC · MATCH · H D A) sharing ONE COL geometry const with the rows, denser label-free price cells. REJECTED: column sorting (round grouping IS the informative ordering in a 13-row rail; Today's Movers already ranks by Δ) and in-rail search (duplicate intent with ⌘K). Registry pulls searched and rejected with logged misses (design/PROVENANCE.md row 34: 21st.dev id-9045, id-5917, shadcn TanStack tables, magicui).
- **One press vocabulary + one tooltip system** (emil-design-eng plans 005/006, commit f6017d6): every pressable on the trade path dips at `active:scale-[0.97]` transform-only (PriceCells, TradePanel chips/Max/submit, FeaturedPanel CTA, onboarding claim, rail pills, Replay); imperceptible 0.99 dips and the last `transition-all` removed; tailwind `future.hoverOnlyWhenSupported` gates all `hover:` variants behind `(hover:hover)`. ONE app-level radix TooltipProvider (skipDelayDuration 300) makes sibling tooltips instant; Tooltip+HoverCard scale in from their trigger via `.pop-content` (radix origin var, var(--duration-fast) var(--ease-out), keyframe on delayed-open ONLY — instant-open deliberately unanimated). The ⌘K menu stays animation-free (keyboard-frequency law). Design-cop: SHIP 12/12 after a round-2 fix pass (44px pills, logged provenance) — design/verdicts/2026-07-15-terminal-craft.md.
- **Desktop Thrashing C is a known, filed gap; the grade gate is "no regression".** MotionScore re-run fresh on the live deploy via the local CLI (`npx motionscore` — free cloud quota exhausted; `?v` cache-busting only busts per-URL): Overall B 75/100 — Desktop B with Thrashing C (unchanged vs 07-14), Mobile A with Thrashing S (up from A after the scoped will-change pass on the hero draw). The desktop C is structural (5 JS scroll listeners, 4 concurrent rAF loops — GSAP/framer/lightweight-charts/ScrollTrigger — plus per-frame text writes in counters); the ScrollTimeline/batching migration is specified in plans/009-desktop-thrash.md, not hacked at 3am. No subscore regressed.

Consequences / VERIFY: a manual `width:`/`resize()` on a chart under autoSize, a second pane in a River container, a new in-rail sort/search, a sub-0.97 press dip, a second TooltipProvider, or an animated ⌘K is a bug. River/motion claims verify on the live deploy only. LAW (owner applies — CLAUDE.md edits are the user's call): design-law section should add "Charts size via lightweight-charts autoSize only (no manual width/resize); River containers are cleared before createChart. Pressables dip at active:scale-[0.97] transform-only; tooltips flow through the ONE app TooltipProvider (.pop-content, delayed-open animation only); the ⌘K menu is never animated."
