# ADR 010 typed event contract anyevent union

Status: accepted 2026-07-07. Context: packages/schema already owned the Envelope contract and topic names, but every `payload` was `z.record(z.unknown())` — untyped. With payloads untyped, each service (engine, ingest, cortex, jobs, web) would hand-shape its own event payloads and they would drift. Session RULE: no service ever defines its own event shape again. Decision: (1) every `Envelope.type` now has exactly ONE zod payload schema in `packages/schema/src/payloads.ts` — odds_tick, goal, card, penalty, kickoff/ht/ft collapsed onto a shared StatusPayload, order, fill, position, halt/reopen collapsed onto a shared HaltReopenPayload, mark reusing the existing Mark, commentary, settled. (2) `packages/schema/src/events.ts` binds type→payload as 15 discriminated-union variants and exports `AnyEvent = z.discriminatedUnion("type", [...])`, plus a `PAYLOAD_BY_TYPE` registry and parse helpers `parseEvent` / `safeParseEvent` / `parsePayload`. (3) TWO-TIER contract: the loose `Envelope` (payload: record(unknown)) is retained ONLY as the bus *transport* type so `packages/bus` stays decoupled from payload shapes; any consumer needing a typed payload calls `parseEvent`, which discriminates on `type` and validates the payload. Both `Envelope` and every `AnyEvent` variant extend a shared `EnvelopeBase`, so envelope fields have one definition. (4) Envelope-level fields (match_id, market_id, source, seq, timestamps) are never duplicated inside payloads — one source per field. (5) `penalty` is in the `EventType` enum but was omitted from the task's payload list; it got a payload so the union stays exhaustive over the enum, and a round-trip test asserts `AnyEvent.options.length === EventType.options.length` to catch any future forgotten variant. Consequences: (+) the RULE is enforced structurally — the only way to add an event is to add a payload + a union variant in packages/schema, guarded by an exhaustiveness test; services import AnyEvent/payload schemas rather than redefining shapes. (+) the discriminated union gives compile-time payload narrowing on `.type`. (−) an event now has two representations (loose Envelope for transport + strict AnyEvent for typed use) that must stay in sync — mitigated by both extending the shared EnvelopeBase. (−) packages/schema gained a vitest devDep to run the round-trip tests (previously test was a no-op echo). No change to bus wire format, topics, or any CLAUDE.md law or hook.
