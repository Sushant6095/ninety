# ADR 009 dev postgres host port 5433

Status: accepted 2026-07-07. Context: fresh-tree first-run bring-up — `docker compose up -d` failed to start Postgres because host port 5432 was already allocated by an unrelated, actively-running local container (openmetadata_postgresql, healthy). Stopping another project's DB is intrusive and out of scope; a `docker-compose.override.yml` can't fix it because Compose concatenates `ports` lists rather than replacing them, so 5432 would still bind. Decision: docker-compose.yml postgres mapping changed `5432:5432` → `5433:5432` (container-internal port stays 5432); apps/api/.env `DATABASE_URL=postgresql://omnipitch:omnipitch@localhost:5433/omnipitch`. Only the host publish port moved — container port and all service-to-service assumptions are unchanged. `.env.example` keeps 5432 as the canonical template default; per-machine `.env` overrides as needed. Consequences: (+) coexists with other local Postgres instances without disruption — `prisma migrate dev` and `pnpm dev` work out of the box on this machine. (−) one-line divergence between the committed compose host port (5433) and `.env.example` (5432); clean-machine contributors can revert to 5432 freely since only the host side changed. No impact on CI, production, or on-chain/settlement paths. For traceability: the same bring-up session also applied mechanical workspace fixes (not decisions) — added root `@types/node` so Node globals resolve, reformatted `prisma/schema.prisma` from invalid compact one-liner syntax to canonical Prisma, stubbed web `lint` to `echo lint` to drop the deprecated interactive `next lint`.
